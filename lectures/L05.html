s5.1 — Classification vs Localisation vs Detection

The three CV tasks on a single image

Why it matters: every later lecture (YOLO, R-CNN, segmentation) is just a more advanced version of one of these three problems. Exam questions love "is this task localisation or detection?"

  • Object Classification — single object in the image. The model must decide which class the object belongs to. No position information is produced.
  • Object Localisation & Classification — still a single object. The model produces both the class label and a bounding box $(b_x, b_y, b_w, b_h)$ around it.
  • Object Detection — multiple objects, possibly of different classes. The model produces a class label and a bounding box for each object. The detector only finds objects of the classes it was trained on (everything else is "background").

Mantra to memorise

  • Classification = what?
  • Localisation = what + where? (single object)
  • Detection = what + where? (multiple objects)
  • Segmentation = what + which pixels? (pixel-wise mask, not a rectangle).

Localisation vs Segmentation — a subtle but exam-favourite distinction

Both produce per-object output but at different granularities. Localisation outputs a rectangle; segmentation outputs a per-pixel mask that follows the actual silhouette of the object.

Tiny taxonomy you should be able to draw on the exam
Task#objectsOutput
Classification1class label
Localisation1class + box $(b_x,b_y,b_w,b_h)$

Pure classification — a CNN with a softmax head

This is the starting point. Take a CNN trained for classification, and we will gradually upgrade its output to also produce a box, then make it run at every position to handle multiple objects.

Note the explicit "None of the above" (or "Background") class — needed because the network must be able to say nothing is here; this prepares the ground for the $p_c$ indicator we add in s5.2.

Localisation — adding a bounding box to the output

Once the network can say "this is a person", we want it to also say where. The trick: extend the output layer with four extra regression outputs $(b_x,b_y,b_w,b_h)$ that describe a normalised box.

Convention used in the slides (top-left of the image is $(0,0)$, bottom-right is $(1,1)$):

  • $b_x, b_y$ — coordinates of the top-left corner of the box, divided by the image width / height.
  • $b_w, b_h$ — width and height of the box, divided by the image width / height.

In the slide: $b_x=0.15$, $b_y=0.1$, $W=0.5$, $H=0.9$ — the person occupies most of the vertical extent but only half the horizontal extent of the image.

s5.2 — The Output Vector $Y$ and the Don't-Care Loss

The unified output vector $Y$

Why it matters: this single vector is the conceptual backbone of every object-detection model in the rest of the course (YOLO especially). Memorise the layout and the symbols.

$$Y = \begin{bmatrix} p_c \\ b_x \\ b_y \\ b_w \\ b_h \\ c_1 \\ c_2 \\ \vdots \\ c_N \end{bmatrix}$$

Where every symbol means:

  • $p_c$ — probability that the patch contains any object of the classes we care about (i.e. not background). Sometimes called the objectness score. Equals $1$ in the training label when an object is present, $0$ otherwise.
  • $b_x, b_y$ — normalised top-left coordinates of the bounding box.
  • $b_w, b_h$ — normalised width and height of the bounding box.
  • $c_1, c_2, \ldots, c_N$ — one-hot (or softmax) class scores for the $N$ classes the network is trained on. Exactly one of them is 1 in the training label when an object is present.

The full length of $Y$ is therefore $\mathbf{1 + 4 + N}$.

Worked example — the slide's "Person" target (3 classes: Person, Cat, Car)

Slide 9: a person in the image, box at normalised coords $(0.2,0.1,0.6,0.9)$. With three classes ordered Person, Cat, Car:

$$Y = [\,1,\;0.2,\;0.1,\;0.6,\;0.9,\;1,\;0,\;0\,]^T$$

The leading $1$ is $p_c$ (object present). The trailing $[1,0,0]$ says class = Person.

The "don't-care" idea: when $p_c = 0$ the box and class entries are meaningless

If the image is just a cat and our classes are {Person, Cat, Car} where Cat is also not in our learned set (i.e. just background), then the network's $b_x,b_y,b_w,b_h,c_1,\dots,c_N$ outputs are undefined — there is no box to predict. Marking them "don't care" is what allows training on negative examples without penalising arbitrary box predictions.

So the labels come in two flavours:

  • Object present ($p_c=1$): full target $[1, b_x, b_y, b_w, b_h, c_1, \ldots, c_N]^T$.
  • No object ($p_c=0$): target $[0, X, X, X, X, X, \ldots, X]^T$ — only the first entry is supervised.

The piecewise loss function $L(\hat Y, Y)$

Why it matters: this is the single equation that you must reproduce on the exam. It is also the reason YOLO can train on images containing no object without going crazy.

Formally (sum-of-squared errors version from the slide):

$$L(\hat Y, Y) = \begin{cases} (\hat p_c - p_c)^2 & \text{if } p_c = 0 \\[6pt] \displaystyle\sum_{i=1}^{1+4+N} (\hat y_i - y_i)^2 & \text{if } p_c = 1 \end{cases}$$

Written out for $N=3$ classes when $p_c=1$:

$$L = (\hat p_c-1)^2 + (\hat b_x-b_x)^2 + (\hat b_y-b_y)^2 + (\hat b_w-b_w)^2 + (\hat b_h-b_h)^2 + (\hat c_1-c_1)^2 + (\hat c_2-c_2)^2 + (\hat c_3-c_3)^2$$

Worked example — slide's "Person" prediction

True $Y = [1, 0.2, 0.1, 0.6, 0.9, 1, 0, 0]^T$, predicted $\hat Y = [1, 0.25, 0.14, 0.5, 0.8, 1, 0, 0]^T$. Because $p_c=1$:

$L = 0^2 + 0.05^2 + 0.04^2 + 0.10^2 + 0.10^2 + 0^2 + 0^2 + 0^2 = 0.0025 + 0.0016 + 0.01 + 0.01 = 0.0241$.

Worked example — slide 12, background image (cat or empty road)

True $Y = [0, X,X,X,X, X,X,X]^T$, predicted $\hat Y = [0.2, 0.25, 0.14, 0.5, 0.8, 1, 0, 0]^T$.

Because $p_c=0$, only the first term counts:

$L = (0.2 - 0)^2 = 0.04$.

The wrong box and class predictions are not penalised — they are "don't care". Otherwise the gradient would chase a phantom box that doesn't exist, and training would diverge.

Common exam traps

  • Forgetting the piecewise structure and summing all 8 squared errors even when $p_c=0$ — wrong. The slide explicitly says "Other values are Don't Care".
  • Confusing $p_c$ with the class probabilities $c_i$. $p_c$ is binary (object vs no object); the $c_i$ pick which object.
  • Forgetting the "Background / None of the above" class is sometimes used instead of the $p_c$ trick. Both approaches exist.
  • Mixing up coordinate conventions: the slide uses top-left $(b_x,b_y)$. YOLO will switch to centre-of-box later — read the question carefully.
Q. With $N=3$ classes and an empty image, write the target $Y$ and the loss when the network outputs $\hat p_c = 0.4$ (everything else garbage).
Answer$Y = [0, X, X, X, X, X, X, X]^T$, $L = (0.4 - 0)^2 = 0.16$. The garbage box/class outputs contribute zero because of the don't-care rule.
Q. The image contains a car. Classes are {Person, Cat, Car} and the box is at normalised $(b_x,b_y,b_w,b_h)=(0.5,0.6,0.3,0.2)$. Write $Y$.
Answer$Y = [1, 0.5, 0.6, 0.3, 0.2, 0, 0, 1]^T$.

Bounding-box normalisation — pixel → $[0,1]$ worked example

Why it matters: nearly every exam problem starts in pixel coordinates and asks you to convert. Keep the conversion automatic.

Given an image of size $W\times H$ (in pixels) with a box at top-left pixel $(x_p, y_p)$ of pixel size $(w_p, h_p)$, the normalised coordinates are:

$$b_x = \frac{x_p}{W}, \quad b_y = \frac{y_p}{H}, \quad b_w = \frac{w_p}{W}, \quad b_h = \frac{h_p}{H}$$

Worked example — car in a 256×256 image, box at (160, 200), size (80, 60)

Image $W=H=256$. With the slide's top-left-corner convention:

  • $b_x = 160/256 = 0.625$
  • $b_y = 200/256 \approx 0.781$
  • $b_w = 80/256 = 0.3125$
  • $b_h = 60/256 \approx 0.234$

With three classes ordered (Person, Cat, Car) and the car present:

$Y = [\,1,\;0.625,\;0.781,\;0.3125,\;0.234,\;0,\;0,\;1\,]^T$.

(If the convention had been box-centre, the centre would have been at pixel $(160+40, 200+30)=(200,230)$ → $b_x=200/256\approx 0.781$, $b_y=230/256\approx 0.898$.)

s5.3 — Sliding Window Detector (Multiple Objects)

The detection problem: multiple objects, unknown positions

Why it matters: until now we assumed one object filling the frame. Real images have many objects of unknown size and location. The sliding-window detector is the simplest brute-force solution and the conceptual ancestor of every modern detector.

The sliding-window idea

Reduce detection to repeated classification. Slide a fixed-size window across the image, crop each window, feed it to a single-object classifier, and record the windows that fire.

Pseudocode — single-scale sliding window

for each window size W_s in {small, medium, large, ...}:
    for y in 0, stride, 2·stride, ... up to H − W_s:
        for x in 0, stride, 2·stride, ... up to W − W_s:
            patch = image[y : y+W_s, x : x+W_s]
            patch = resize(patch, classifier_input_size)
            ŷ = classifier(patch)            # outputs Y = [pc, bx, by, bw, bh, c1..cN]
            if ŷ.p_c > threshold:
                record(x, y, W_s, class = argmax(c1..cN))
apply non-maxima suppression over all recorded boxes

Multi-scale by window size or by image pyramid

Cars near the camera look big; cars far away look tiny. A single window size cannot catch both, so we sweep many sizes.

Why the two are equivalent Scaling the image by factor $s$ and using a window of size $w$ "sees" the same content as keeping the image and using a window of size $w/s$. So you can either grow the window or shrink the image; computational costs differ but detections agree.

Per-window classification — three ways to build the classifier

Whatever you choose, the contract is the same: input = a fixed-size patch, output = an objectness score and (optionally) a class. The lecture lists three options, classical → modern.

Speeding up: three techniques from the slides

Running a CNN once per window for thousands of windows per image is wildly slow. The slides outline three "tricks" — they later form the backbone of the OverFeat / YOLO speed-ups.

[1] Replace fully-connected layers with 1×1 convolutions

Why a 1×1 convolution can replace a fully-connected layer A fully-connected layer is just a matrix multiply: out[i] = Σ W[i,j] · in[j]. If the input tensor before FC has shape $H_f\times W_f\times C$ and the FC has output size $K$, the FC weights are a matrix of shape $K\times (H_f W_f C)$. Replace it with a Conv2D using $K$ filters of spatial size $H_f\times W_f$ and $C$ input channels — at this spatial resolution the two are mathematically identical. Subsequent FC layers become $1\times 1$ convolutions in the same way. The win comes at inference time: the same conv head now works on larger inputs and produces a spatial grid of predictions in one shot.

[2] Apply the CNN to the whole image at once — share computation across windows

[3] Filter factorisation — stack small filters instead of one big one

The receptive-field arithmetic Stacking $k$ convolutions of spatial size $f\times f$ (stride 1, no padding) gives a receptive field of $1 + k(f-1)$ pixels on the input. For $f=5, k=5$: $1 + 5\cdot 4 = 21$. ✓ So five 5×5 layers exactly cover the 21×21 footprint of the single big filter.

Overlapping detections → Non-Maxima Suppression

Because the windows overlap, the same object will fire many times at slightly different positions and scales. We need a post-processing step that keeps only the best box per object.

Putting it all together — sliding window for detection + localisation

Final picture: combine the sliding window of section s5.3 with the output vector $Y$ of section s5.2. Every window produces a full $Y$ and we keep the windows where $p_c$ is high.

Worked example — slide 46 output vectors

Three detected positions:

  • Left window (object): $[1, 0.3, 0.1, 0.6, 0.9, 1, 0, 0]^T$ → Person.
  • Right window (background): $[0, X,X,X,X, X,X,X]^T$ → discarded.
  • Bottom-centre window (object): $[0.7, 0.2, 0.2, 0.6, 0.9, 1, 0, 0]^T$ → Person, but borderline confidence.

After NMS, overlapping high-$p_c$ boxes for the same person collapse into one.

Limitations of sliding window → motivation for YOLO (L07)

Why it matters: every limitation here is a feature YOLO sets out to fix. Knowing them prepares you for L07.

  • Very, very slow. Even with shared computation and factorised filters, multiple window sizes and multiple aspect ratios mean many passes per image.
  • Many passes. Multi-scale ⇒ many window sizes ⇒ image scanned many times.
  • Stride / precision trade-off. Small stride = many windows (slow). Large stride = miss the exact pixel location of an object (loss of localisation precision).
  • Fixed-size classifier. Each crop must be resized before feeding the classifier — distorts aspect ratios for non-square objects (taxi vs bicycle).
  • Brute-force aspect-ratio sweep. A "person" is tall and skinny; a "car" is short and wide. Either we use many window aspect ratios or we accept distortion.
  • No end-to-end training. The classifier was trained on object/background patches; it never saw the geometry of where objects appear together.

What YOLO will fix (preview of L07)

  • Replace the brute-force window sweep with a single forward pass that divides the image into a grid and emits $Y$ for every cell at once.
  • Train end-to-end on the full image — the network learns spatial context.
  • Predict multiple boxes per cell with different aspect ratios via "anchor boxes".
  • Still ends with Non-Maxima Suppression.

Common exam traps

  • "Sliding window cannot detect multiple objects." — False. It can; it's just slow.
  • "A larger stride speeds up and improves precision." — False; stride trades speed for precision.
  • "Replacing FC with 1×1 conv changes the math." — False; it's an algebraic identity. The benefit is variable input size + spatial reuse, not new accuracy.
  • "Image pyramid and multi-window-size are different strategies." — They are mathematically equivalent.
Q. A 200×200 image is scanned by a 50×50 window with stride 10. How many window positions are evaluated (one scale only)?
AnswerNumber of starts along one axis = $\lfloor (200 - 50)/10 \rfloor + 1 = 16$. Total positions = $16\times 16 = 256$.
Q. Why does the slide-36 example use stride 1 and not stride 2?
AnswerStride 1 maximises localisation precision; once we have the shared-computation trick of slide 40, stride 1 costs essentially the same as stride 2, so we may as well use it.

Practice quiz — Lecture 5

1 (MCQ). Which task produces a per-pixel mask rather than a rectangle?
a) Classification   b) Localisation   c) Detection   d) Segmentation
Answerd) Segmentation. Localisation and detection produce rectangular bounding boxes.
2 (MCQ). For 3 classes (Person, Cat, Car), what is the length of the output vector $Y$?
a) 3   b) 4   c) 7   d) 8
Answerd) 8 = 1 ($p_c$) + 4 (box) + 3 (classes).
3 (T/F). When $p_c = 0$ in the target, the loss still penalises the box coordinates $b_x, b_y, b_w, b_h$.
AnswerFalse. Those entries are "don't care"; only $(\hat p_c - p_c)^2$ contributes.
4 (T/F). Sweeping multiple window sizes is mathematically equivalent to sweeping a single window over an image pyramid.
AnswerTrue. Scaling the image by $s$ and using window $w$ ≡ keeping the image and using window $w/s$.
5 (Short). Write the target vector $Y$ for an image containing one car, with classes ordered (Person, Cat, Car) and a normalised box $(0.4, 0.5, 0.2, 0.3)$.
Answer$Y = [1, 0.4, 0.5, 0.2, 0.3, 0, 0, 1]^T$.
6 (Numeric). A 256×256 image has a car with top-left corner at pixel (160, 200) and pixel size (80, 60). Using the slide's top-left convention, write the normalised box.
Answer$b_x = 160/256 = 0.625$, $b_y = 200/256 \approx 0.781$, $b_w = 80/256 = 0.3125$, $b_h = 60/256 \approx 0.234$.
7 (Numeric). True $Y = [1, 0.5, 0.5, 0.4, 0.4, 0, 1, 0]^T$ (cat). Predicted $\hat Y = [0.9, 0.55, 0.45, 0.4, 0.35, 0.1, 0.8, 0.1]^T$. Compute the sum-of-squared-errors loss.
AnswerComponents squared: $0.1^2 + 0.05^2 + 0.05^2 + 0^2 + 0.05^2 + 0.1^2 + 0.2^2 + 0.1^2 = 0.01 + 0.0025 + 0.0025 + 0 + 0.0025 + 0.01 + 0.04 + 0.01 = 0.0775$.
8 (Scenario). Sliding window with stride 16 on a 320×320 image, fixed 64×64 window, single scale. (a) How many positions? (b) Name one drawback of using stride 16 instead of stride 4.
Answer(a) Starts along one axis = $\lfloor (320-64)/16 \rfloor + 1 = 17$ → $17\times 17 = 289$ positions. (b) Reduced localisation precision: a true object whose top-left lies between two stride steps cannot be matched exactly; the predicted box can be off by up to 16 pixels.
9 (Short). Why is the explicit "Background" / "None of the above" class useful in a classification network, and how does $p_c$ replace it in the localisation network?
AnswerA classification softmax always picks some class; without a "None" class it would force a label onto background patches. In the localisation network the binary $p_c$ output acts as that toggle — if $p_c$ is low, the patch is background and the class scores are ignored.
10 (Identify). In the loss $L = (\hat p_c - p_c)^2 + (\hat b_x - b_x)^2 + (\hat c_1 - c_1)^2$, which terms become "don't care" when $p_c=0$?
Answer$(\hat b_x - b_x)^2$ and $(\hat c_1 - c_1)^2$ (and the analogous $b_y, b_w, b_h, c_2, \ldots$). Only $(\hat p_c - p_c)^2$ is kept.
