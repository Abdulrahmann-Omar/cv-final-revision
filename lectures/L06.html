s6.1 — Regression Loss for Bounding Boxes: MAE vs MSE

Setting the scene: which detector is best?

Why it matters: every modern detector regresses 4 box coordinates. To train it we need a loss that says how wrong the predicted box is, and to evaluate it we need a metric of geometric closeness. This lecture pins down both.

A bounding box is parameterised either as (x, y, w, h) (top-left + width & height) or as (x₁, y₁, x₂, y₂) (top-left + bottom-right). Both encodings appear all term — switch fluently.

Regression loss — absolute (MAE / L1) vs squared (MSE / L2) distance

Why it matters: this is literally the loss we use on the four corner coordinates before any fancy IoU loss.

Let box 1 (ground truth) have corners $(X_{11},Y_{11})$ and $(X_{12},Y_{12})$, and box 2 (prediction) have corners $(X_{21},Y_{21})$ and $(X_{22},Y_{22})$.

Absolute (L1 / MAE) distance:

$$L_{\text{MAE}} = |X_{11}-X_{21}| + |Y_{11}-Y_{21}| + |X_{12}-X_{22}| + |Y_{12}-Y_{22}|$$

Squared (L2 / MSE) distance:

$$L_{\text{MSE}} = (X_{11}-X_{21})^2 + (Y_{11}-Y_{21})^2 + (X_{12}-X_{22})^2 + (Y_{12}-Y_{22})^2$$

Generically, for any prediction $\hat y$ and target $y$:

$$L_{\text{MAE}}=|y-\hat y|, \qquad L_{\text{MSE}}=(y-\hat y)^2$$

Worked numeric — GT (50,50,150,150), prediction (55,60,140,160) Corner errors: $|50-55|+|50-60|+|150-140|+|150-160| = 5+10+10+10 = 35$ (MAE).
Squared: $25+100+100+100 = 325$ (MSE).

Which is better — MSE or MAE?

  • MAE (L1) — linear penalty. Gradient is $\pm 1$ everywhere. Robust to outliers: a single huge error doesn't dominate. Slow near the minimum (gradient never shrinks → it can wobble).
  • MSE (L2) — quadratic penalty. Gradient $= 2(y-\hat y)$, grows linearly with the error → faster convergence when errors are large (which the slide highlights). Heavily punishes outliers: one bad prediction with error 10 contributes 100, vs 10 for MAE.

Slide's takeaway: "as the difference increases, the gradient of squared error increases more rapidly than the absolute error → faster convergence" — provided outliers are not pathological.

Smooth-L1 / Huber loss — the practical compromise (used in Fast R-CNN, SSD…) Smooth-L1 behaves like MSE near zero (so the gradient shrinks as you approach the optimum, killing the L1 wobble) and like MAE for large errors (so a single bad outlier cannot blow up the gradient): $$\text{Smooth-L1}(x)=\begin{cases}0.5\,x^2 & |x|<1 \\ |x|-0.5 & \text{otherwise}\end{cases}$$ Best of both worlds; that's why every classic detector uses it for box regression.

The hidden flaw — corner regression ignores object scale

Why it matters: this single observation justifies the rest of the lecture (IoU, GIoU, DIoU, CIoU). If you grasp the slide-6 trick, you understand why we abandon raw L1/L2 on coordinates.

  • Left: large object — prediction is "close" to ground truth.
  • Right: small object — same Δ in pixels but the prediction is "far away" visually.
  • L1/L2 loss is identical in both cases because it only adds up coordinate differences. The visual quality differs hugely.

Conclusion: we need a loss that depends on the geometry of the overlap, not on raw pixel differences. Enter IoU.

Common exam traps

  • "MSE is always better than MAE" — only when outliers are rare. With a mislabelled box, MSE can ruin training.
  • "MAE has zero gradient near the minimum" — wrong: gradient is $\pm 1$, never zero. That's why training plateaus need a learning-rate decay or Smooth-L1.
  • "L1/L2 corner regression accounts for object size" — false; it treats a 5-pixel shift on a tiny object the same as on a billboard.
Q. A prediction has corner error 10 px. Compare MSE vs MAE contributions.
AnswerMAE contributes $|10|=10$; MSE contributes $10^2=100$. MSE punishes outliers 10× harder here.
Q. Which loss should you reach for if your labels contain a few mis-clicks?
AnswerMAE (or Smooth-L1) — robust to a handful of huge outliers.

s6.2 — Intersection over Union (IoU) and IoU-based Losses

What IoU measures

Why it matters: IoU is the standard evaluation metric for detection accuracy in this course (and across the field — COCO, PASCAL VOC, KITTI…). It also seeds an entire family of losses.

Definition. For two axis-aligned boxes $A$ and $B$:

$$\boxed{\;\mathrm{IoU}(A,B) \;=\; \frac{|A\cap B|}{|A\cup B|}\;}$$

It is a scalar in $[0,1]$ — 0 means no overlap, 1 means perfect alignment. IoU is a measure of overlap between two bounding boxes.

The closed-form computation with $(x_1,y_1,x_2,y_2)$

Let box 1 be $(X_{11},Y_{11},X_{12},Y_{12})$ and box 2 be $(X_{21},Y_{21},X_{22},Y_{22})$.

Intersection rectangle — clip from both sides:

$$\text{Inter}_W=\max\!\left(0,\;\min(X_{12},X_{22})-\max(X_{11},X_{21})\right)$$

$$\text{Inter}_H=\max\!\left(0,\;\min(Y_{12},Y_{22})-\max(Y_{11},Y_{21})\right)$$

$$\text{Inter area}=\text{Inter}_W \cdot \text{Inter}_H$$

(The slide writes it without the $\max(0,\cdot)$ — you must add it yourself when the boxes don't overlap, otherwise the formula returns a negative number.)

Union area (inclusion–exclusion):

$$\text{Union}=(X_{12}-X_{11})(Y_{12}-Y_{11})+(X_{22}-X_{21})(Y_{22}-Y_{21})-\text{Inter area}$$

Finally:

$$\mathrm{IoU}=\frac{\text{Inter area}}{\text{Union area}}$$

Worked numerical example — A=(50,50,150,150), B=(100,100,200,200)

Both boxes are 100×100, area = 10 000 each.

Intersection corners:

  • $\max(50,100)=100,\;\max(50,100)=100$ → top-left of intersection $(100,100)$.
  • $\min(150,200)=150,\;\min(150,200)=150$ → bottom-right $(150,150)$.

Intersection size: $50\times 50 = 2{,}500$.

Union: $10{,}000 + 10{,}000 - 2{,}500 = 17{,}500$.

$$\mathrm{IoU}=\dfrac{2500}{17500}=\dfrac{1}{7}\approx 0.143.$$

Verdict: well below the typical 0.5 threshold → this is not a valid detection match.

Second worked example — perfect overlap and disjoint cases

Perfect overlap: A=B=(0,0,100,100). Inter = Union = 10 000. IoU = 1.

Disjoint: A=(0,0,10,10), B=(50,50,60,60). $\min(10,60)-\max(0,50)=10-50=-40$ → clip to 0. Inter = 0, IoU = 0.

Visual catalogue of IoU values

Key insight from the bottom-left panel: even when one box is entirely inside the other, IoU is only 1 if both have the same area. Containment ≠ match.

Properties of IoU

  • Non-negativity: $\mathrm{IoU}\geq 0$ — areas can't be negative.
  • Symmetry: $\mathrm{IoU}(A,B)=\mathrm{IoU}(B,A)$ — the metric doesn't care which is "prediction" and which is "GT".
  • Scale insensitivity / invariance: if you blow up both boxes by the same factor, IoU stays the same. This fixes the headache from slide 6: IoU treats small and large objects fairly.

Bonus property: $\mathrm{IoU}\in[0,1]$. Upper bound comes from $|A\cap B|\leq|A\cup B|$.

IoU-based loss function

Why it matters: the natural way to turn "higher IoU is better" into something a gradient-descent optimiser can minimise.

$$\boxed{\;\mathcal{L}_{\mathrm{IoU}} = 1 - \mathrm{IoU}\;}$$

Range $[0,1]$ — 0 when boxes coincide, 1 when they are disjoint. Why a direct IoU loss is desirable:

  • Independent of box scale — solves the small-object problem from s6.1.
  • Captures geometry (overlap), not raw pixel offsets.
  • Single scalar that already balances $x_1,y_1,x_2,y_2$ contributions automatically.

The catch — zero overlap kills the gradient

  • If $A\cap B=\emptyset$ then $\mathrm{IoU}=0$ and $\mathcal{L}_{\mathrm{IoU}}=1$ — constant.
  • Constant loss ⇒ zero gradient ⇒ no weight update.
  • This is the typical situation early in training when initial predictions miss the object entirely.

Question raised by the slide: how do we enhance the IoU-based loss? Answer: GIoU, DIoU, CIoU.

[2] GIoU — Generalised IoU

Why it matters: gives a useful gradient even when the boxes are disjoint.

Let $C$ be the smallest convex (rectangular) box that contains both $A$ and $B$.

$$\mathrm{GIoU} = \mathrm{IoU} - \frac{|C - (A\cup B)|}{|C|}$$

$$\mathcal{L}_{\mathrm{GIoU}} = 1 - \mathrm{IoU} + \frac{|C - (A\cup B)|}{|C|}$$

Range: $\mathrm{GIoU}\in[-1,1]$, so $\mathcal{L}_{\mathrm{GIoU}}\in[0,2]$.

Intuition: the extra term penalises empty area inside the smallest enclosing box — minimising it pulls the prediction toward the ground truth even when they don't overlap.

Problem. GIoU ignores the distance between the centres of the boxes — three different predictions can share the same GIoU even though one is clearly best.

[3] DIoU — Distance IoU

Let $d$ = distance between centres of the two boxes, and $C$ = the smallest enclosing box (same as in GIoU); its diagonal length is $c$.

$$\mathrm{DIoU}=\mathrm{IoU}-\frac{d^2}{c^2}\qquad \mathcal{L}_{\mathrm{DIoU}}=1-\mathrm{IoU}+\frac{d^2}{c^2}$$

This punishes predictions whose centres are far from the ground-truth centre, even when IoU and GIoU are unchanged. The normalisation by $c^2$ keeps the penalty scale-invariant.

[4] CIoU — Complete IoU

Define the orientation / aspect-ratio penalty:

$$v = \frac{4}{\pi^2}\left(\arctan\frac{w^{gt}}{h^{gt}} - \arctan\frac{w}{h}\right)^{\!2}$$

which lives in $[0,1]$ because the angle range is $[0,\pi/2]$ (we normalise by $(\pi/2)^2$).

$$\mathrm{CIoU}=\mathrm{IoU}-\frac{d^2}{c^2}-\alpha\,v,\qquad \alpha=\frac{v}{(1-\mathrm{IoU})+v}$$

$$\mathcal{L}_{\mathrm{CIoU}}=1-\mathrm{IoU}+\frac{d^2}{c^2}+\alpha\,v$$

So CIoU jointly accounts for overlap (IoU), centre distance ($d^2/c^2$), and aspect ratio ($v$). Used in YOLOv4 onward — empirically the best of this family.

Other variants (mentioned briefly)

The slide lists four further variants — know the names; the details are not examined here:

  • EIoU — Efficient IoU.
  • Focal-EIoU — adds focal weighting.
  • SIoU — SCYLLA-IoU, adds an angle term.
  • WIoU — Wise-IoU.

Common exam traps

  • "IoU loss always has a useful gradient" — false. When boxes are disjoint, IoU = 0 and the loss is constant.
  • Forgetting the $\max(0,\cdot)$ on the intersection width/height — gives negative areas and wrong IoU.
  • "Two boxes with the same IoU are equally good" — GIoU/DIoU/CIoU exist precisely because that's not true.
  • Mixing up $(x,y,w,h)$ with $(x_1,y_1,x_2,y_2)$ — keep a consistent encoding throughout a problem.
  • Claiming IoU is not scale-invariant — it is (multiplicative scaling cancels in numerator and denominator).
Q. Compute IoU between A=(0,0,2,2) and B=(1,1,3,3).
AnswerInter corners $(\max(0,1),\max(0,1),\min(2,3),\min(2,3))=(1,1,2,2)$. Inter area = 1. Each box area = 4. Union = 4+4−1 = 7. IoU = 1/7 ≈ 0.143.
Q. For A=(0,0,4,4) and B=(0,0,2,2), is IoU = 1?
AnswerNo. Inter = 4, Union = 16+4−4 = 16, IoU = 4/16 = 0.25. Containment ≠ perfect match.
Q. Two boxes are disjoint. What is $\mathcal{L}_{\mathrm{IoU}}$? Is the gradient useful?
AnswerLoss = 1 (constant). Gradient = 0 → useless for learning. GIoU/DIoU fix this.

s6.3 — Non-Maximum Suppression (NMS)

Quick review: a detector's output per box

Every predicted box carries an objectness probability $P_c$ (also called confidence). Higher $P_c$ ⇒ more confident this box really contains an object. Each class also has its own probability $c_i$.

The problem — overlapping detections of the same object

So the rule "keep only the most confident box" works for one object but throws away other objects entirely. We need a per-cluster rule. That's NMS.

The NMS algorithm

Slide's six steps verbatim:

  1. Discard all bounding boxes with $P_c < 0.6$ (low probability to be an object).
  2. Select the box with the highest $P_c$.
  3. Calculate IoU between the selected box and all other remaining boxes.
  4. Discard boxes with IoU > 0.5 (heavily overlapping with the selected box).
  5. Repeat 2–4 excluding the previously selected predicted box.
  6. Stop when there is no box left to discard.

Note: $P_c < 0.6$ and $\mathrm{IoU} > 0.5$ are user-defined thresholds — pick them per task.

Pseudocode equivalent:

Input : list of boxes B with scores S, IoU threshold T
Output: filtered list of detections D

# step 1 — confidence filter
B = { b in B : score(b) >= P_min }    # e.g. P_min = 0.6

D = [ ]                               # detections kept
while B is not empty:
    m = argmax_{b in B} score(b)      # step 2 — highest score
    D.append(m)
    B.remove(m)                       # selected boxes are excluded
    for each remaining b in B:        # step 3 — pairwise IoU
        if IoU(b, m) >= T:            # step 4 — discard heavy overlaps
            B.remove(b)               # (e.g. T = 0.5)
return D                              # step 6 — stop when B empty
Why this is "non-maximum suppression" At each iteration we keep the local maximum of the confidence and suppress (remove) every neighbour that overlaps it heavily. We never suppress the maximum itself — hence "non-maximum suppression".

Worked walk-through from the slides

Hand trace — 5 boxes, T = 0.5

Boxes (with scores):

  • $b_1$: score 0.9, A=(0,0,10,10)
  • $b_2$: score 0.85, B=(1,1,11,11) — IoU with $b_1$ ≈ 0.68
  • $b_3$: score 0.7, C=(2,2,12,12) — IoU with $b_1$ ≈ 0.47
  • $b_4$: score 0.95, D=(50,50,60,60) — disjoint from cluster
  • $b_5$: score 0.4, E=(51,51,61,61)

With $P_{\min}=0.6$: drop $b_5$. Remaining $\{b_1,b_2,b_3,b_4\}$.

Round 1: argmax is $b_4$ (0.95). Keep $b_4$. IoU($b_1$, $b_4$)=0, IoU($b_2$, $b_4$)=0, IoU($b_3$, $b_4$)=0 — none discarded.

Round 2: argmax is $b_1$ (0.9). Keep $b_1$. IoU($b_2$, $b_1$)≈0.68 > 0.5 → drop $b_2$. IoU($b_3$, $b_1$)≈0.47 < 0.5 → keep.

Round 3: argmax is $b_3$ (0.7). Keep $b_3$. Nothing remains.

Output: $\{b_4, b_1, b_3\}$.

Notice: $b_3$ survives because its IoU with $b_1$ is below the threshold — that may be a problem when two real cars are parked next to each other (see the trap below).

NMS is greedy — and that has consequences

  • Greedy: never reconsiders earlier choices. Cheap, $O(n^2)$ in the worst case.
  • Sensitive to threshold T:
    • $T$ too low (e.g. 0.3) → over-suppression: two real adjacent objects whose boxes overlap a lot are merged into one detection (failure case the lecture warns about).
    • $T$ too high (e.g. 0.9) → under-suppression: duplicate boxes survive on the same object.
    • Typical default: $T=0.5$.
  • Failure case (two nearby objects): if two real cars are bumper-to-bumper, their best boxes may have IoU above $T$. NMS deletes one of them and you lose a true positive.
  • Soft-NMS (research variant): instead of removing a heavily-overlapping box, decay its score by a function of the IoU. The slide doesn't cover it explicitly, but it's the standard remedy for the dense-object failure case.

Common exam traps

  • "NMS uses two thresholds." — Yes: the confidence cut $P_{\min}$ (step 1) and the IoU cut $T$ (step 4). Don't conflate them.
  • "NMS picks the box with highest IoU." — No. It picks the box with the highest confidence, then uses IoU only to remove competitors.
  • "NMS is run once per image." — Standardly run per class: a car box and a pedestrian box never compete, even if they overlap.
  • "NMS is differentiable." — No, it's a discrete greedy algorithm; that's why end-to-end detectors (e.g. DETR) try to eliminate it.
  • Forgetting to exclude already-selected boxes in step 5 — the slide highlights "selected boxes are excluded".
Q. Which threshold value of $T$ in NMS would increase false positives (duplicate boxes)?
AnswerA high $T$ (close to 1). With $T=0.9$ only nearly-identical boxes get suppressed, so most duplicates survive.
Q. Why is NMS usually run per class?
AnswerSo an overlapping car box and pedestrian box don't suppress each other — they're different objects.
Q. NMS is "non-maximum suppression". What is being kept, and what is suppressed?
AnswerKept: the local maximum of the confidence. Suppressed: every neighbouring box whose IoU with that maximum exceeds $T$.

Practice quiz — Lecture 6

1 (Numerical IoU). Boxes A=(50,50,150,150) and B=(100,100,200,200). Compute IoU.
AnswerInter corners: $(\max(50,100), \max(50,100), \min(150,200), \min(150,200))=(100,100,150,150)$ → 50×50 = 2 500. A area = B area = 100×100 = 10 000. Union = 10 000+10 000−2 500 = 17 500. IoU = 2 500 / 17 500 ≈ 0.143.
2 (Numerical IoU). A = (0, 0, 4, 4), B = (2, 2, 6, 6). IoU = ?
AnswerInter = (2, 2, 4, 4) → 2×2 = 4. Areas 16+16. Union = 32 − 4 = 28. IoU = 4/28 = 1/7 ≈ 0.143.
3 (Conceptual MAE/MSE). State one situation where MAE is preferable to MSE and one where MSE wins.
AnswerMAE wins when labels contain outliers (MSE would let one bad sample dominate). MSE wins when errors are small and you want fast convergence near the optimum (gradient grows linearly with error, vanishes at the optimum).
4 (NMS walk-through). Boxes (score, x1,y1,x2,y2): b1=(0.95, 0,0,10,10); b2=(0.9, 1,1,11,11); b3=(0.5, 2,2,12,12); b4=(0.85, 100,100,110,110). $P_{\min}=0.6$, T=0.5. Run NMS step by step.
Answer Step 1: drop b3 (score 0.5 < 0.6). Remaining {b1, b2, b4}.
Round 1: argmax = b1 (0.95). Keep b1. IoU(b1, b2)= 81/(100+100−81)=81/119 ≈ 0.68 > 0.5 → drop b2. IoU(b1, b4)=0 → keep.
Round 2: argmax = b4 (0.85). Keep b4. Nothing left.
Output: {b1, b4}.
5 (T/F). $\mathcal{L}_{\mathrm{IoU}} = 1 - \mathrm{IoU}$ provides a useful gradient even when prediction and ground truth are disjoint.
AnswerFalse. When IoU = 0 the loss is the constant 1, so the gradient vanishes. That's the motivation for GIoU.
6 (MCQ). Which of the following IoU variants takes aspect ratio into account?
a) IoU   b) GIoU   c) DIoU   d) CIoU
Answerd) CIoU — adds the $\alpha v$ term where $v\propto(\arctan(w^{gt}/h^{gt})-\arctan(w/h))^2$.
7 (Scenario). Your detector outputs three heavily-overlapping boxes on the same pedestrian and a single confident box on a separate dog. You run NMS with $T=0.5$ globally over all classes. Why is this a bad idea?
AnswerNMS should be applied per class. Running it globally lets the pedestrian box potentially suppress an overlapping dog box (or vice versa), which is wrong: they are different objects and should both be kept.
8 (Short). Why is IoU scale-invariant?
AnswerIf both boxes are scaled by factor $k$, both the intersection area and union area scale by $k^2$, and the ratio is unchanged. So small and large objects are evaluated on the same footing.
