Q1¬∑A ‚ÄĒ Ten True/False statements [10 Marks]

Solution

For T/F items, the trick is usually one small clause that's deliberately wrong. Read the sentence twice; if any one piece (a word like "always", a wrong direction of effect, or a swapped concept) is off, the whole thing is False.
#Statement (paraphrased)T/FWhy
1A non-linearly separable set can be made linearly separable in another (higher-dim) feature space.TrueExactly the kernel-trick / lifting idea behind SVMs.
2When Gaussian filtering, kernel size is the most important because it defines the scale.FalseIt's ŌÉ that defines the scale. Kernel size just has to be large enough (‚Čą6ŌÉ) to fit the bell.
3Adaboost handles outliers by lowering the weight of repeatedly-misclassified points.FalseAdaboost increases the weight of misclassified points ‚ÄĒ outliers actually become problematic for Adaboost.
4Averaging smooths images by assuming noise is independent.TrueAveraging cancels zero-mean independent noise; that's the whole intuition.
5Gaussian filter is edge-preserving.FalseIt blurs everything including edges. Bilateral / median are edge-preserving.
6For Sobel, correlation and convolution give the same result.FalseSobel is not symmetric (it's anti-symmetric), so flipping the kernel changes the sign.
7k-NN has no explicit training step.TrueIt's a lazy learner ‚ÄĒ training is just storing the data.
8Mean-shift with the same window size but different starts always converges to the same solution.FalseDifferent starting points climb to different local modes.
9For Gaussian denoising, the noise magnitude decreases as ŌÉ decreases.FalseThe opposite ‚ÄĒ bigger ŌÉ = more blur = more noise suppression.
10Removing a support vector makes the SVM margin increase.FalseSupport vectors define the margin; removing one normally widens the candidate margin in one direction, but the new margin from re-solving usually shrinks overall because a constraint that was pinning it is gone. Standard taught answer: False ‚ÄĒ the margin generally decreases (or stays the same).

Q1¬∑B ‚ÄĒ Brief questions i‚Äďvi [10 Marks]

i. What is template matching? How can it be done with filters?

Template matching = sliding a small reference patch over the image and recording, at every position, "how similar are we?" The similarity can be normalised cross-correlation, sum-of-squared-differences, etc.
A template (small image) is convolved/correlated with the input image. Peaks in the response map indicate where the template matched best. Implementation: use the template directly as a filter kernel and compute correlation.

ii. Arithmetic operation to sharpen an image (example using filters) [2 Marks]

Sharpen = "original + edges". The arithmetic op is addition (image + high-pass version of it). Equivalent kernel: a Laplacian-with-+1-centre.
Sharpened = Original + Laplacian(Original). Example 3√ó3 mask:

iii. Two methods for deciding whether two feature descriptors match [2 Marks]

  1. Threshold on distance (e.g. Euclidean or SSD) ‚ÄĒ if it's below T, declare a match.
  2. Lowe's ratio test ‚ÄĒ keep the match only if (nearest-neighbour distance) / (2nd-nearest distance) < 0.8. This rejects ambiguous matches.

iv. How many point correspondences are needed to fit a 2D translational model? [1 Mark]

A 2D translation has only 2 unknowns (tx, ty). Each correspondence gives 2 equations. So one point pair already gives 2 equations for 2 unknowns.
1 correspondence (the absolute minimum).

v. K-means: pixels of similar colour in disjoint regions belong to different clusters ‚ÄĒ what features?

If you cluster on colour alone, all pixels of the same colour end up in one cluster no matter where they are. To split by region you must add position.
Use a 5-D feature vector (R, G, B, x, y) ‚ÄĒ colour together with pixel coordinates. The spatial term pulls pixels of similar colour but distant location into different clusters.

Q1¬∑B (vi) ‚ÄĒ RANSAC pseudo-code fill-in

The two missing blocks

RANSAC's loop has two phases per iteration: (a) test every point against the current model, growing the consensus set if it fits within tolerance t; (b) at the end, if the consensus set is bigger than d, re-fit on it and keep it if it beats the best one so far.
FOR every data point not in CurrentSet
    IF this point fits CurrentModel with error < t THEN
        add it to ConsensusSet
    END IF
END FOR
IF #(ConsensusSet) > d
    re-fit BetterModel on ConsensusSet
    measure its error on ConsensusSet
    IF this error < best error so far
        BestModel := BetterModel
    END IF
END IF
Inner loop = count inliers. Outer if = re-fit and update best model. After all iterations, return BestModel.

Q2¬∑A¬∑B¬∑C ‚ÄĒ Harris detector / SIFT / Grabcut

A. Harris detector for corner detection ‚ÄĒ main steps [3 Marks]

Corners = where the image looks "different" in both directions when you shift a window. Harris quantifies this with the second-moment matrix M of image gradients and reads its two eigenvalues.
  1. Compute image gradients Ix, Iy (e.g. with Sobel).
  2. For every pixel form the 2√ó2 matrix M = w ‚ąó [[Ix¬≤, IxIy], [IxIy, Iy¬≤]] where w is a Gaussian window.
  3. Compute the response R = det(M) ‚ąí k¬∑trace(M)¬≤ with k ‚Čą 0.04‚Äď0.06.
  4. Threshold R and apply non-max suppression ‚ÄĒ surviving pixels are corners.

Parameters: gradient operator size, window ŌÉ for w, k, response threshold, NMS radius.

B. SIFT descriptor [3 Marks]

After a keypoint is detected, SIFT builds a small "fingerprint" describing the local gradient pattern around it, in a way that survives rotation, scale and brightness changes.
  1. Take a 16√ó16 patch around the keypoint, rotated to the keypoint's dominant orientation (rotation invariance).
  2. Split it into a 4√ó4 grid of 4√ó4 cells. In each cell build an 8-bin gradient orientation histogram weighted by gradient magnitude and a Gaussian.
  3. Concatenate the 16 histograms into a 128-D vector and L2-normalise (illumination invariance).

ii. SIFT is invariant to (any three of):

Scale ¬∑ Rotation ¬∑ Illumination (brightness/contrast) ¬∑ Small viewpoint/affine changes ¬∑ Translation.

C. Grabcut [4 Marks]

Grabcut is the "draw a rectangle around the object" cut-out method. It refines an initial guess with iterated graph-cut.
  1. What image features? Pixel colour (RGB / Lab).
  2. Initial segmentation? User draws a rectangle: everything outside = sure background, everything inside = "possibly foreground".
  3. Model representation? A Gaussian Mixture Model (GMM) per class ‚ÄĒ one GMM for foreground colours, one for background colours (typically 5 components each).
  4. Graph partitioning technique? Min-cut / max-flow on the pixel graph (Boykov‚ÄďKolmogorov).
Harris, SIFT descriptor details, and Grabcut are partly outside this year's syllabus (SIFT keypoint detection is in L04; descriptor stage less so; Grabcut not at all).

Q3¬∑B ‚ÄĒ The operator ‚ąá¬≤hŌÉ(x; y) [6 Marks]

i. General name & purpose

‚ąá¬≤ applied to a Gaussian hŌÉ gives the famous "Mexican-hat" filter ‚ÄĒ the Laplacian-of-Gaussian (LoG). Convolving it with an image gives strong responses at blobs and edges at scale ŌÉ.
Name: Laplacian-of-Gaussian (LoG). Purpose: blob / edge detection at a chosen scale ŌÉ (and used inside DoG-based SIFT).

ii. What does g(x,y) = 0 correspond to in I(x,y)?

Zero-crossings of the second derivative happen exactly where the first derivative has an extremum ‚ÄĒ i.e. at edges.
Pixels where g = 0 (zero-crossings of the LoG response) correspond to edge locations in I.

iii. Significance of ŌÉ. Increase ŌÉ ‚áí more or fewer zero-crossings?

ŌÉ is the scale: bigger ŌÉ blurs more, so only large-scale features survive.
Increasing ŌÉ ‚áí fewer zero-crossings ‚ÄĒ fine-scale edges get smoothed away, only the strong/long edges remain.

iv. Which is preferable: ‚ąá¬≤[hŌÉ ‚ąó I] or [‚ąá¬≤hŌÉ] ‚ąó I, and why?

Convolution is associative and linear. Both forms give the same result, but the second one lets you precompute the LoG kernel once and convolve with the image directly ‚ÄĒ that's a single convolution.
Prefer (b): ‚ąá¬≤hŌÉ(x;y) ‚ąó I(x;y). One convolution with a precomputed LoG kernel is far cheaper than first blurring, then taking the Laplacian numerically.

Q3¬∑C ‚ÄĒ Bayesian classification [4 Marks]

i. Name & meaning of the three terms

Bayes' rule has the same three pieces every time ‚ÄĒ posterior, likelihood, prior ‚ÄĒ divided by the evidence. Memorise the names.
SymbolNameMeaning in words
P(Ck | x)PosteriorHow probable class Ck is after seeing the observation x. The quantity we want.
P(x | Ck)Likelihood (class-conditional density)How likely we would observe x if the true class were Ck. Learned from training data.
P(Ck)PriorBelief about class Ck before seeing the data (e.g. base rates).

ii. Decision rule that minimises misclassification

To minimise the chance of being wrong, pick the class that has the largest posterior probability. The evidence P(x) is the same for every class so it can be dropped.
MAP rule: assign x to class k* where k* = argmaxk P(Ck|x) = argmaxk P(x|Ck) ¬∑ P(Ck).

B. Explanation of the Hough algorithm used in (A)

Each edge pixel "votes" for every line that could possibly pass through it. The line that the most edge pixels agree on becomes the winner.
  1. Discretise (r, őł) into a 2-D accumulator.
  2. For each edge pixel, sweep őł and compute r = x cos őł + y sin őł. Increment H[r,őł].
  3. After all pixels, the bin with the highest vote count is the dominant line; (rm, őłm) = argmax H.
  4. Convert (r, őł) back to the line in image space.

C. Types of object tracking methods (draw & explain)

Object trackers fall into three classical families: point, kernel/template, and silhouette/contour.
FamilyIdeaExamples
Point trackingRepresent the object as one or a few points; predict next position with a motion model and match across frames.Kalman filter, particle filter, optical-flow point tracking.
Kernel / appearance trackingRepresent the object as a template (bounding box + appearance model); search the next frame for the best match.Mean-shift tracker, KLT, correlation filters.
Silhouette / contour trackingTrack the object's outline or full mask; update its shape frame-by-frame.Active contours (snakes), level-sets, shape matching.
Modern deep-learning trackers (Siamese networks, ByteTrack, etc.) extend these classical categories.
