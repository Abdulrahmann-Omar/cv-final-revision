Question 1 — Background subtraction & feature invariance [16 Marks]

Part A — Two ways to estimate the background image B(x, y, t)

A "background image" is what the scene would look like with no moving objects. You either grab one clean frame, or you average many frames so the moving stuff washes out. Each trick has a clear downside — that's what the question wants you to name.
MethodIdea (one line)Disadvantage
1. Static reference frame Take a single frame of the empty scene before any object enters, and use that as B(x, y, t) for every later frame. It is wrong the moment anything in the scene changes — lighting shifts, a chair gets moved, the camera nudges. Any permanent change leaks in as fake foreground.
2. Running average / median of recent frames Update B(x, y, t) every frame as a weighted blend of the previous background and the new frame (or take the temporal median over a window). If an object stays still for a while it gets absorbed into the background; if an object moves very slowly it can leave a "ghost" trail; you also need a learning rate that's neither too sticky nor too fast.

Part B — Feature invariance, detectors and descriptors

i. What does "invariant to a transformation X" mean? (with two examples)

It means you can apply X to the image and the same feature still pops out at the same place with the same description. Two examples of X: translation (shifting the image left/right) and rotation (turning the image).

ii. Why is invariance a desirable property?

If a feature dies the moment the camera tilts or the object shifts, it's useless for matching across two photos of the same scene.
So the same physical point in the world produces the same feature vector across different views — which is what makes matching, tracking and recognition work in real conditions.

iii. Detector vs descriptor (one example each)

DetectorDescriptor
Job Find where the interesting points are. It scans the image and flags locally distinctive structures (corners, blobs, ...). Once a location is given, summarise what it looks like as a compact vector that throws away irrelevant info but keeps enough to match.
ExampleHarris corner detectorSIFT descriptor

iv. How to make SIFT rotation invariant

The trick is to give every keypoint its own "north" — a canonical orientation — and describe the patch relative to it. Then a rotated copy of the same patch ends up with the same descriptor.
  1. Around the keypoint, compute the gradient direction at each pixel of the local window.
  2. Build a 36-bin histogram of those directions (10° per bin), weighted by gradient magnitude and by a Gaussian centred on the keypoint.
  3. The highest peak of that histogram is the keypoint's canonical orientation.
  4. Rotate the window so that orientation points "up", then compute the descriptor on the rotated window.
Because the descriptor is always computed in the keypoint's own frame, the same physical patch produces the same descriptor no matter how the image is rotated.

Question 2 — Meanshift tracking & CV term defs [22 Marks]

Part A — Mean-shift for object tracking, step by step

Mean-shift is "follow the centre of mass". Build a colour (or feature) histogram of the object in frame 1, then in every later frame slide a search window towards the place that matches that histogram best. No model retraining, just iterative centre-of-mass updates.
  1. Initialise (frame 0). User (or detector) puts a box around the target. Compute the colour histogram of the pixels inside that box — this is the target model.
  2. New frame arrives. Start the search window at the last known location.
  3. Back-project. For every pixel in the new frame, look up its colour in the target histogram and replace the pixel by its histogram weight. Pixels that look like the target become bright, the rest become dark.
  4. Compute the centre of mass of that weighted map inside the search window.
  5. Shift the window so its centre lands on that centre of mass.
  6. Repeat 4 – 5 until the window stops moving (convergence) — usually 2 – 5 iterations.
  7. The converged centre is the new (x, y) of the object; carry it as the start for the next frame.

Part B — Define + explain each term

i. Second moment matrix

It's the 2 × 2 matrix of squared/mixed image gradients summed over a small window. Its eigenvalues tell you: both small → flat region, one large → edge, both large → corner.
Use: the Harris corner detector builds the second-moment matrix for every pixel; the "corner-ness" score is computed from its eigenvalues to flag corners as keypoints. (This is exactly the official answer.)

ii. Gaussian pyramid

Repeatedly blur with a Gaussian then downsample by 2 ⇒ a stack of progressively smaller, smoother copies of the same image.
Use: scale-invariant feature detection (SIFT keypoints) and coarse-to-fine search. By detecting features at every level of the pyramid, you find the same feature whether the object is photographed close-up or far away. Also used to pre-filter for sub-sampling without aliasing.

iii. Bag of Words (BoVW)

Treat an image like a paragraph: extract local descriptors (SIFT), cluster them into "visual words", then represent the image by the histogram of which words appear. Throws away spatial layout, keeps composition.
Use: image classification / object recognition — a classifier (e.g. SVM) is trained on these word-histograms. The official key phrases it as "a way to simplify object representation as a collection of their subparts for purposes such as classification."

iv. Kernel trick

Instead of explicitly mapping data into a high-dimensional space, replace every inner product ⟨x, x′⟩ by a kernel K(x, x′) that equals that inner product in the higher space. You get the power of the non-linear lift without ever computing the lift.
Use: non-linear SVM classifiers for object recognition — RBF/poly kernels let a linear SVM separate classes that are not linearly separable in pixel/feature space. Same trick powers kernel PCA and kernel ridge regression.

v. Dynamics of a moving object

A model of how the object's state (position, velocity, ...) evolves between consecutive frames — e.g. constant velocity, constant acceleration.
Use: visual tracking with a Kalman filter / particle filter. The dynamics give a prediction of where the object will be in the next frame; the observation (detection) then corrects that prediction. This is what makes trackers smooth and robust to short missed detections.

📚 Harris / second-moment → L04; pyramid → L04; BoVW + kernel + SVM → L05.

Question 3·A — GrabCut for foreground–background segmentation [8 Marks]

i. What features can be used for segmentation?

"Features" in segmentation just means anything per-pixel that helps tell foreground from background — and the most powerful signal in a still photo is colour.
Per-pixel colour (typically RGB or Lab), optionally augmented by texture/gradient or spatial position. GrabCut as taught uses colour only.

ii. How is the initial segmentation done?

The user gives a rough hint — usually a single rectangle around the object — and that hint seeds the first labelling.
The user draws a bounding rectangle around the object. Everything outside the rectangle is marked as definite background; everything inside the rectangle is marked as probable foreground. (Optional brush strokes can mark sure-foreground/sure-background pixels.)

iii. What model represents the two classes?

You need a soft probabilistic model — "how likely is this colour to come from the object vs the background?" GrabCut uses a Gaussian mixture for each class.
A Gaussian Mixture Model (GMM) per class — usually 5 Gaussians for the foreground colours and 5 for the background colours. Each iteration re-fits the two GMMs on the pixels currently assigned to each class.

iv. What technique partitions the graph?

Once every pixel is a graph node connected to a "foreground" terminal and a "background" terminal (with weights from the GMMs and edge weights to neighbours), cutting the graph at minimum total weight gives the best labelling.
Graph cut — specifically the min-cut / max-flow algorithm. GrabCut iterates GMM re-fit → graph-cut → repeat, until the labelling stops changing.

Question 3·B — Track a face by skin colour [7 Marks]

Solution — full pipeline (offline prep + per-frame loop)

When the cue is "skin colour" and the target is "one face in every frame", the textbook answer is: build a skin-colour model offline, and at runtime back-project + mean-shift to lock onto the centre. State the inputs and outputs clearly — examiners reward that.

Inputs and outputs

  • Input: a sequence of video frames It(x, y) (one face in each, facing camera).
  • Output: for each frame, the (x, y) coordinates of the face centre.

Offline preparation (done once)

  1. Collect skin patches from a few face photos (or the first frame).
  2. Convert to a colour space that separates chrominance from brightness — typically HSV or YCbCr — so lighting changes have less effect.
  3. Build a skin-colour histogram (or a small GMM) from those patches. This is the target model.

Initialisation (frame 0)

  1. Run a face detector once (e.g. Viola–Jones) or back-project the skin histogram on frame 0 and take the largest skin blob.
  2. Place a search window around that blob. This is the starting box.

Per-frame tracking loop

  1. Convert the new frame to HSV/YCbCr.
  2. Back-project the skin histogram → a probability map where bright pixels = skin-coloured.
  3. Run mean-shift on that map, starting from the previous frame's window, to find the new centre of mass.
  4. Output (x, y) = converged window centre.
  5. Carry the window into the next frame.
In one sentence: train a skin-colour histogram offline → for each frame back-project it and run mean-shift → return the converged window centre as (x, y).

Question 4 — Two iterations of AdaBoost [12 Marks]

The three equations first (fill the top table)

Memorise these three formulas — AdaBoost is just running them in a loop:
QuantityEquationPlain-English
Error rate errm errm = (sum of weights of misclassified points) / (sum of all weights) How much weight is currently sitting on the points this learner got wrong.
Voting power αm αm = ½ · ln((1 − errm) / errm) How much this round's learner gets to "vote". Small err ⇒ big α.
Weight update wi wi ← wi · exp(αm) if i misclassified, wi · exp(−αm) if i correct; then renormalise so all wi sum to 1. Misclassified points get up-weighted, correctly classified points get down-weighted.
