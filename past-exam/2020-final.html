Q1·1 — Matching two images at different scales

Answer: b) SIFT

"Harris" = corners only, not scale-invariant. "SIFT" = built on a scale-space pyramid → it finds the same feature whether the object is near or far. Whenever the question mentions different scales, the answer is SIFT.

📚 L04 — SIFT / scale-space.

Q1·2 — RANSAC sample size for a translational model

Answer: a) one

RANSAC needs as many points as the degrees of freedom (DoF) of the transform.
  • Translation only (Δx, Δy) → 2 unknowns → solvable from 1 point pair (each pair gives 2 equations).
  • Similarity → 2 pairs · Affine → 3 pairs · Homography → 4 pairs.

Q1·3 — Harris corner from the eigenvalues

Answer: c) λ1 and λ2 are both large

The second-moment matrix M has eigenvalues that measure how much the image gradient varies in two perpendicular directions:
  • both small → flat region
  • one large, one small (λ₁ ≫ λ₂) → edge
  • both large → corner ✓

Q1·4 — Label vector for a single-car image

Answer: b) y = [1, 0.3, 0.7, 0.3, 0.3, 0, 1, 0]

Read y = [pc, bx, by, bh, bw, c1, c2, c3]. The first slot is the "is there an object?" flag. After that come the four box coords (centre x, centre y, height, width). After that comes the one-hot class vector.
  • Image clearly contains a car → pc = 1, so (a) and (d) are out.
  • Car is class 2 → one-hot must be (0, 1, 0), so (c) is out (it has class 1 selected).
That leaves (b).

Q1·5 — IoU of a 2×2 box and a 2×3 box with overlap 1×1

Answer: b) 1/9

IoU = (overlap area) / (union area), and union = A + B − overlap. Always subtract the overlap once or you double-count it.

Overlap = 1·1 = 1.
AreaA = 2·2 = 4, AreaB = 2·3 = 6.
Union = 4 + 6 − 1 = 9.
IoU = 1 / 9.

Q1·6 — Siamese network for face verification

Answer: d) different input images / same parameters

The whole point of a Siamese net is to map different images through the same encoder, then compare embeddings. Same weights = the two branches are literally one network shared twice. Different weights would break the whole metric idea.

Q1·7 — What does the homography matrix model?

Answer: d) perspective

A 3×3 homography H captures the most general 2D-to-2D transform we use: rotation + scale + shear and the projective vanishing-point part. The last row of H is what does perspective — that's what differentiates it from an affine matrix.

📚 L05 — Homography / projective transforms.

Q1·8 — Which detector for person discrimination?

Answer: b) HOG (answer key marks it "out of scope?" — but Dalal & Triggs HOG is the classic pedestrian descriptor)

HOG = Histogram of Oriented Gradients = the descriptor designed exactly for human silhouettes. Harris finds corners. SIFT finds blobs at various scales. Only HOG captures the shape of the whole body.

Q1·9 — Up-sampling inside a CNN

Answer: d) deconvolution

In the answer key Dr. Khattab picks (d) "deconvolution" — which is the name commonly used in the slides for the upsampling op. Strictly speaking the same operation is also called transpose convolution (c). Both names refer to the learnable up-sampling block used in FCN-style decoders. "Pooling" and "strided conv" go the opposite way — they downsample.

Q1·10 — How many boxes survive NMS?

Answer: c) 5 (the answer key says "I see nothing"; counting the visible scores answers it)

NMS in two steps. Step 1 — drop everything with p ≤ 0.4. Step 2 — within each remaining cluster, keep only the highest-confidence box and suppress neighbours with IoU > 0.5.

Boxes printed on the image and their scores:

  • tree 0.46 ✓ (above 0.4, unique tree)
  • tree 0.74 ✓ (above 0.4, unique tree)
  • motorcycle 0.58 ✓ (above 0.4, unique class)
  • pedestrian 0.98 ✓ (above 0.4, unique)
  • car 0.62 ✓ (highest among the overlapping cars)
  • car 0.73 — overlaps strongly with 0.62; both are cars, only one kept. Since 0.73 > 0.62, 0.73 stays, 0.62 dies. Net: still one car here.
  • car 0.26 ✗ (below threshold, discarded)
Surviving boxes: tree·0.46, tree·0.74, motorcycle·0.58, pedestrian·0.98, car·0.73 → 5 boxes.

Q1·11 — YOLO output volume size (19×19, 20 classes, 5 anchors)

Answer: a) 19×19×45

YOLO output = Grid × Grid × ( 5·B + C ).
5 = (pc, bx, by, bh, bw) per anchor. C is shared per cell (not per anchor) in the YOLOv1-style formula used in the answer key.
19 × 19 × (5·5 + 20) = 19 × 19 × 45.

Note — YOLOv2 onwards uses (5+C)·B per anchor (= 5·B + B·C), which would give 19×19×125. The slide course/answer key uses the v1 formula, so the answer is (a).

Q1·12 — Intuition behind detecting a corner

Answer: c) high

If you slide the window over a corner, any direction you move produces a large change in pixel values. That's the whole definition. Flat region → low. Edge → high in one direction only. Corner → high in every direction.

Q1·13 — Triplet loss margin α

Answer: c) α > 0

α is the margin. We want anchor-positive distance to be smaller than anchor-negative distance by at least α. If α = 0, mapping every image to the same vector would trivially make the loss 0 — useless. We need α strictly positive to force separation.

📚 Siamese / Triplet loss.

Q1·14 — Features for k-means on colour respecting location

Answer: c) Both colour and location

If you cluster on colour alone, two red patches on opposite sides of the image land in the same cluster — that's the opposite of what we want. Concatenating (R, G, B, x, y) as the feature forces pixels far apart in space into separate clusters even if their colour matches.

Q1·15 — Which metric is not a MOT accuracy metric?

Answer: b) Frame per second (fps) (out of scope per answer key)

MT, MOTP, average IoU all measure quality of the tracks. fps measures speed, not accuracy. The odd-one-out is fps.

Q1·16 — MeanShift convergence depends on …

Answer: d) window size

MeanShift slides a window of radius h up the local density. Bigger h ⇒ smoother density ⇒ fewer modes; smaller h ⇒ more modes. So the size of the kernel directly decides which mode the trajectory converges to. The initial position only changes which mode you find, not whether you converge.

Q1·17 — When is a detection a false negative?

Answer: d) no bounding box produced for the object

"False negative" = the object exists but the detector missed it. IoU thresholds (a, b, c) define true-positive vs false-positive; a complete miss is the only condition that names a false negative.

Q1·18 — RANSAC sample count with 30% outliers, p = 80%

Answer: d) 3 (the key writes 1.33 → ≈ 2; rounded up to the next integer that beats it gives 2 in their writing. The closest MCQ option above is d = 3 to be safe.)

The textbook formula is $N > \dfrac{\log(1-p)}{\log\bigl(1-(1-e)^{s}\bigr)}$
where e is the outlier fraction, s is the sample size needed for one fit, and p is the required confidence.

e = 0.3, s = 1 (translational model, see Q1·2), p = 0.8.

N > log(1 − 0.8) / log(1 − 0.7) = log(0.2) / log(0.3) ≈ 1.33 → round up to 2.

The answer key writes "≈ 2"; only (d) 3 from the choices is ≥ 2 if we read "at least" strictly. The exam's MCQ choices are ambiguous here — both (c) 2 and (d) 3 are defensible.

Q1·19 — What problem does Hierarchical LK solve?

Answer: c) When the motion is no longer small (out of scope per answer key)

Plain Lucas–Kanade assumes a tiny motion (a few pixels). Hierarchical LK runs LK at coarse resolution first (where a large displacement is small), then refines down the pyramid — fixing the large-motion failure mode.

Q1·20 — Bag-of-Words k

Answer: a) representative of the number of visual words in the dataset

Each cluster centre in BoW is a "visual word". So k = how many visual words you'd like in your vocabulary. It is not tied to number of classes (c) or number of features (d) directly.

Q1·21 — Good feature descriptor properties

Answer: d) all of the above

A good descriptor must be (1) repeatable across views, (2) distinctive so we can match it, and (3) local so partial occlusions don't kill it. SIFT was designed to hit all three.

📝 Question 2 — Tracking by colour & cell segmentation (13 Marks)

Q2·A — Track a car with a hue histogram + MeanShift (out of scope per answer key)

Pipeline

MeanShift tracking = "follow the colour blob". You make a back-projection map (likelihood that each pixel matches the target colour histogram), then slide a window up the density of that map.
  1. Build the target model from the first frame: convert the bounding box around the car to HSV, take the H channel, build a 16-bin histogram, and normalise it to a probability distribution.
  2. For every new frame: convert the whole frame to HSV.
  3. Back-projection — replace each pixel by the histogram value of its hue bin. Bright pixels = "looks like the car colour".
  4. Run MeanShift on the back-projection map, starting from the previous frame's window position. The window iteratively recenters on the local maximum of the colour likelihood.
  5. Update the tracker position to the converged window centre. (Optionally re-estimate the histogram every K frames to adapt to lighting drift — CamShift does this.)
  6. Repeat for every subsequent frame.
The hue-only histogram makes the tracker robust to brightness and shadow changes — that's exactly why HSV is used here instead of RGB.

Q2·B — Cell image labelling (semantic → instance)

(i) Name of the task

Per-pixel binary label (cell / not-cell) over the whole image, with no separation of individual cells, is the textbook definition of semantic segmentation.
Answer: Semantic Image Segmentation.

(ii) CNN architecture

The standard answer Dr. Khattab gives is the Deconvolution Network (or its sibling, U-Net / FCN). Two halves: encoder shrinks the image, decoder grows it back to original size.
  • Input: the raw H×W×3 microscope image.
  • Core (encoder): a stack of conv + ReLU + max-pool blocks (e.g. like VGG-16) → feature map of size 7×7 with rich semantic channels.
  • Core (decoder): mirror stack of deconv (= transpose conv) + unpooling, doubling spatial size at every stage, until back to H×W.
  • Output layer: 1×1 conv → 1 channel + sigmoid (or 2 channels + softmax) → per-pixel cell probability map.
  • Train with per-pixel cross-entropy against the doctor's binary masks.

(iii) What the doctors want instead

If the doctors need to tell individual cells apart, semantic segmentation isn't enough — we need a separate mask per cell. That's instance segmentation. The classic architecture is Mask R-CNN: detection + per-instance mask.
Answer: Instance Image Segmentation.

(iv) Required relabelling of training data

For Mask R-CNN you need (a) a bounding box per cell and (b) a per-cell binary mask (instead of a single full-image mask). The doctors' current target images need to be edited so every individual cell is its own labelled object.

📝 Question 3 — Pedestrian detector design (10 Marks)

Q3 — Compare R-CNN / Fast / Faster / YOLO + tracking errors

(i) Four-detector comparison table

Two axes drive the whole story: region proposals (do we still rely on selective search?) and number of CNN passes per image (one or 2000?).
Region proposals # ConvNets Class head Speed
R-CNN Selective search 2,000 (one per proposal) per-class linear SVMs Slow (∼47 s/image)
Fast R-CNN Selective search 1 (shared feature map) Softmax Fast (∼2 s/image)
Faster R-CNN RPN (learned, end-to-end) 1 Softmax Faster (∼0.2 s/image)
YOLO none — grid + anchors 1 Softmax (per cell in v1) Real-time (≈ 45 fps)

(ii) Tracker error types (out of scope per answer key)

A multi-object tracker has to associate detections from frame t to frame t + 1. Two famous things that break the association:
  • Occlusion → the detector skips the object for several frames → tracker must use track continuation: keep a "ghost" track alive on its predicted (e.g. Kalman) location and re-attach when the object reappears. Otherwise we get an ID switch or a fragmented track.
  • Two pedestrians crossing → two detections, one near each other → naive association by IoU swaps the IDs. We need an appearance model (colour/feature similarity, à la Deep SORT) on top of motion to prevent the ID-switch error.

📝 Question 4 — PCA / Eigenfaces (8 Marks)

Q4 — Face recognition with PCA (out of scope per answer key)

(i) Training procedure

PCA on faces = "Eigenfaces" (Turk & Pentland, 1991). The trick: every 100×100 face is one 10,000-dim point; PCA picks the few directions of largest variance among faces, and we project to that small subspace.
  1. Stack each training image as a 10,000-dim column vector. Get a 10,000 × N data matrix.
  2. Centre: subtract the mean face μ from every column.
  3. Compute the covariance matrix (or the much cheaper N × N Gram trick) and take its top k eigenvectors. These are the eigenfaces.
  4. Project every training image into the k-dim eigenface subspace and store the (label, projected vector) pairs.

Testing procedure

  1. Take the query image; subtract the mean face.
  2. Project it onto the same k eigenfaces → a k-dim vector.
  3. Find the nearest training projection (e.g. by L2 distance / cosine similarity).
  4. (Optional rejection step) If the distance to the closest match is above a threshold, declare "unknown".

(ii) Strengths and weaknesses

Strengths Weaknesses
  • Massive dimensionality reduction (10,000 → tens).
  • Fast nearest-neighbour search in low-d space.
  • Captures the global "what does a face look like" structure with very little supervision.
  • Eigenfaces also give a compact image representation — good for compression / visualisation.
  • Holistic appearance-based — extremely sensitive to illumination, pose, expression, alignment. A shadow can dominate the principal components.
  • Linear — cannot model non-linear manifold structure of faces (e.g. profile vs frontal).
  • Not discriminative: PCA picks variance, not class-separation (LDA / FisherFaces beats it on identification).
  • Adding a new person ideally requires retraining (or at least re-projecting).
