SC · Question 1 — 14 True / False statements (14 marks)

Answer key (T/F + correction)

For T/F items the trick is to spot a single wrong word — usually a swapped algorithm name ("SIFT" ↔ "feature detector", "LK" ↔ "image rotation"), or a flipped property ("up-sampling" instead of "down-sampling"). Replace the wrong word and the statement becomes correct.

SC · Question 2 — Colored points + two filtered cameras (8) & Mean Shift (7)

A.i — Which points does each camera see?

A pure red object reflects only red, so a green filter blocks it entirely. White is a mix of every wavelength, so both filters pass it. Net effect: each camera sees its own color plus all the whites.
$C_R$ (red filter): sees the 5 red points + 2 white points = 7 points.
$C_G$ (green filter): sees the 5 green points + 2 white points = 7 points.

A.ii — Enough correspondences for a translation?

The two cameras share only the white points. Count shared points: those are your usable correspondences. Then check whether that count covers the number of unknowns.
Yes. The 2 white points are visible in both cameras, so we get 2 matches. A translation-only homography has only 2 unknowns ($t_x, t_y$), and 2 matches give 2 equations ⇒ uniquely solvable.

A.iii — Algorithm to compute good correspondences (RANSAC)

"Set of good correspondences in the presence of outliers" = RANSAC. Always state the 4-step loop and the 3 hyper-parameters $s, T, N$.
  1. Sample $s$ correspondences at random (the minimum needed to fit the model — e.g. 4 for a full homography, 2 for translation).
  2. Solve for the model parameters using just that sample.
  3. Score by counting how many of the remaining matches fall within distance $T$ of the model (= the inliers).
  4. Repeat $N$ times, then keep the model with the most inliers and refit on its inlier set.

Parameters:

  • $s$ — minimum points to fit. Use the smallest number the model allows (≥ 2 for translation, ≥ 4 for homography).
  • $T$ — distance threshold. Pick $T$ so that an inlier passes with high probability (≈ 0.95).
  • $N$ — number of iterations. Pick $N$ so that, with probability $p$ (≈ 0.99), at least one sample is outlier-free: $N = \lceil \log(1-p)/\log(1-(1-e)^s) \rceil$.

B.i — Main steps of Mean Shift segmentation

Remember Mean Shift as "climb to the nearest peak in feature space, then merge nearby peaks." The only parameter is the window (bandwidth) size.
  1. Find features per pixel (color, gradients, texture, position, …).
  2. Initialise a search window centred on each feature point.
  3. Compute the mean of the data inside the window (centroid of the points it covers).
  4. Shift the window centre to that mean. Repeat 3–4 until convergence (the window stops moving).
  5. Merge windows that converge to the same peak/mode → one cluster per peak.

Parameter to set: window (bandwidth) size.

B.ii — Sensitivity to the window size

Very large window → fast but inaccurate; different real objects can collapse into one cluster.
Very small window → slow but accurate; but noise pixels can each become their own "object".

B.iii — Why Mean Shift beats K-means for image segmentation

  1. No shape assumption. K-means assumes roughly spherical clusters; Mean Shift just follows the density.
  2. Not sensitive to outliers — outliers don't shift the mode much.
  3. No need to fix K in advance — the number of modes (clusters) is discovered automatically.

SC · Question 3 — Eight short-answer items + sketch a network (20 marks)

A.1 — What is a Gaussian pyramid? Two applications

A Gaussian pyramid is the cascade of successively blurred & downsampled copies of an image (each level = half the resolution of the previous).
Uses: (1) multi-scale feature search in SIFT / DoG, (2) coarse-to-fine optical flow in hierarchical LK (handles big motions).

A.2 — Direction in which optical flow cannot be reliably estimated

Aperture problem: a 1-D edge only lets you measure the flow component perpendicular to the edge. The component along the edge is invisible.
Optical flow cannot be reliably estimated along the direction of an edge (the tangent direction). You can only recover the normal-to-edge component.

A.3 — Why use SIFT instead of "simpler features"?

SIFT keypoints are robust to scale, rotation, illumination and small viewpoint changes. Each keypoint has a window that compares many pixels at once → produces a high-dimensional vector very unlikely to collide between unrelated patches. Whereas raw pixel patches break under any transformation.

A.4 — Two main loss terms of region-based detectors (R-CNN family)

(i) Classification loss (softmax / cross-entropy over the C+1 classes) and (ii) Bounding-box regression loss (smooth-L1 on the 4 box offsets). Faster R-CNN adds two more inside the RPN head: objectness + RPN bbox regression.

A.5 — One example of a region-based CNN that succeeded the Faster R-CNN object detector

Mask R-CNN — Faster R-CNN + a parallel mask branch + RoIAlign instead of RoIPool.

A.6 — Explain the RPN in Faster R-CNN

RPN = a small fully-convolutional network that slides over the backbone feature map and, at every spatial location, predicts (a) an objectness score and (b) box-coordinate offsets for each of $k$ pre-defined anchors. Its output is a ranked list of object proposals (regions of interest) fed downstream to the classifier and bbox refiner.

A.7 — Typical YOLO output as a single-shot detector

YOLO replaces "propose then classify" with "predict everything in one forward pass". Each grid cell directly outputs box + objectness + class scores.
$\text{Output} = S \times S \times (5 \cdot B + C)$, where:
  • $S$ = grid size,
  • $5 = (dx, dy, dh, dw, \text{conf})$ per bounding box,
  • $B$ = number of bounding boxes (anchors) per cell,
  • $C$ = number of classes to predict.

A.8 — Modified output for class-aware anchors

$\text{Output} = S \times S \times (B \cdot (5 + C))$ — each anchor box gets its own class probabilities (this is how YOLOv2+ allows multiple classes per cell).

B — Sketch a network that does both semantic and instance segmentation on driving images

"Both semantic + instance" = Panoptic / Mask R-CNN style. Branch 1 keeps the FCN-style per-pixel softmax for "road/sky/lane" (stuff classes). Branch 2 is Mask R-CNN — RPN → RoIAlign → (class, box, mask) per object (things classes).
Input: 3-channel RGB image.
Core: shared CNN backbone (ResNet/FPN) → splits into two heads:
  • Semantic head (FCN): encoder–decoder up to H×W×K_stuff, softmax per pixel.
  • Instance head (Mask R-CNN): RPN → RoIAlign → 3 sub-heads producing class label, refined box, and 28×28 binary mask per detected object.
Output: (a) a per-pixel class map (stuff: road, sky…) and (b) a list of object instances with masks (things: car, pedestrian, traffic-sign).

SC · Question 4 — Car & person re-identification (Siamese + BoVW)

A — Car-arrival detector at your garage (Siamese network)

"One reference image of my car + decide for any new frame: is this it or not?" — classic one-shot verification task. Solution: a Siamese network trained with a contrastive (or triplet) loss to produce small distance for "same" and large distance for "different".

A.i — Architecture

  • Two identical CNN towers (shared weights, e.g. ResNet/VGG) that each map an image to an embedding $f(x)$.
  • The two embeddings are compared with a distance: $d(x^{(1)}, x^{(2)}) = \lVert f(x^{(1)}) - f(x^{(2)}) \rVert_2^2$.
  • At training time we feed pairs (or triplets) with a same/different label and minimise a contrastive (or triplet) loss.

Triplet loss (the version used in the answer key):

$L(A, P, N) = \max\bigl(\lVert f(A) - f(P) \rVert_2^2 - \lVert f(A) - f(N) \rVert_2^2 + \alpha,\; 0\bigr)$
where $A$ = anchor (your car), $P$ = positive (same car), $N$ = negative (different car), $\alpha$ = margin.

A.ii — Training procedure

  1. Inputs. Triplets: anchor + positive (same car, different lighting/angle) + negative (a different car).
  2. Learning objective. Minimise distance(anchor, positive); maximise distance(anchor, negative); push apart by at least margin $\alpha$.
  3. Loss. Triplet (or contrastive). Stop when validation accuracy plateaus.

B — Find every photo containing yourself in a big trip album

"One template + thousands of cluttered photos, occluded faces, scale changes" → exact template matching fails. Use a Bag-of-Visual-Words (BoVW) pipeline: descriptors → vocabulary → histogram → k-NN.
  1. For every photo (including your reference photo):
    1. Detect & describe local features with SIFT.
    2. Learn a visual vocabulary by clustering all descriptors (k-means → "visual words").
    3. Quantise each photo's descriptors against the vocabulary.
    4. Represent each photo by the histogram of visual words (frequencies).
    5. Normalise each histogram into a feature vector.
  2. For the reference template, run k-NN over the photo vectors using a similarity metric (cosine / chi-squared) → returns the top matches.
  3. For each candidate whose distance is below a chosen threshold → add it to the "I appear in" subset.

B.i — Brightness constancy constraint and its assumptions

Three letters: BCC. The whole optical-flow derivation rests on these.
Brightness constancy: $I(x, y, t) = I(x + u, y + v, t + 1)$. The same physical surface point has the same intensity in the next frame.
Assumptions:
  • Small motion between consecutive frames (so we can Taylor-expand).
  • Constant illumination across the two frames (no shadow/light change).
  • Spatial smoothness — neighbouring pixels move together (the Lucas–Kanade addition).

B.ii — Optical-flow algorithm in matrix form

Apply Taylor expansion → $I_x u + I_y v + I_t = 0$ (one equation, two unknowns — aperture problem).
Lucas–Kanade adds the smoothness assumption: every pixel in a small window shares the same $(u,v)$. Stack the equations over the $n$ pixels in the window:
$A \begin{bmatrix} u \\ v \end{bmatrix} = -b$, with $A = \begin{bmatrix} I_x^{(1)} & I_y^{(1)} \\ \vdots & \vdots \\ I_x^{(n)} & I_y^{(n)} \end{bmatrix}$, $b = \begin{bmatrix} I_t^{(1)} \\ \vdots \\ I_t^{(n)} \end{bmatrix}$.
Least-squares solution: $\begin{bmatrix} u \\ v \end{bmatrix} = (A^\top A)^{-1} A^\top (-b)$.

B.iii — Non-linear motion → linearised algorithm

"Non-linear motion" = big displacement between frames → the first-order Taylor expansion breaks. Fix: iterate.
  1. Build a Gaussian pyramid of both frames.
  2. Solve the linear LK at the coarsest level (motion is small there).
  3. Up-sample the flow, warp the next frame using the current estimate, and re-solve LK at the next pyramid level. Repeat down to full resolution.
  4. Each iteration updates the motion estimate $(u, v) \mathrel{+}= (\delta u, \delta v)$ → this is exactly the linearised / iterative LK algorithm.

Credit · Question 2 B — Harris corner detector (5.5 marks)

Q2 A is identical to SC Q2 A — see the SC solution above. Only Q2 B (Harris) is new.

i — Harris error equation

Idea: shift a small window by $(u,v)$ and see how much the image inside changes. A flat region barely changes; an edge changes only in one direction; a corner changes in every direction.
$E(u, v) = \sum_{(x,y) \in W} \bigl[I(x+u, y+v) - I(x, y)\bigr]^2$

ii — Second-moment matrix & how it picks good corners

Taylor-expand → $E(u,v) \approx \begin{bmatrix} u & v \end{bmatrix} M \begin{bmatrix} u \\ v \end{bmatrix}$ where $M = \begin{bmatrix} \sum I_x^2 & \sum I_x I_y \\ \sum I_x I_y & \sum I_y^2 \end{bmatrix}$.
A good corner is found where both eigenvalues $\lambda_1, \lambda_2$ of $M$ are large — i.e. the local intensity changes a lot in two independent directions.
One eigenvalue large → edge. Both small → flat region. Both large → corner.

iii — Main steps of the Harris algorithm

  1. Compute derivatives $I_x, I_y$ at every pixel (e.g. with Sobel).
  2. Compute second-moment matrix $M$ per pixel, smoothed by a Gaussian over a small window.
  3. Compute corner response $R = \det(M) - k \cdot (\operatorname{tr} M)^2$, with $k \in [0.04, 0.06]$.
  4. Threshold $R$ — keep only pixels with $R$ above some value.
  5. Non-maximum suppression on $R$ to get one pixel per actual corner.

Credit · Question 3 — Semantic-seg network & SOT vs MOT (10 marks)

A — Sketch a CNN to find the road by semantic segmentation

"Find the road" = label every pixel as road / not-road = semantic segmentation. Use a fully-convolutional encoder–decoder (FCN / U-Net / SegNet / DeconvNet).
Input: 3-channel road image (e.g. 224×224×3).
Core (encoder–decoder):
  • Convolution / encoder path: a stack of conv + ReLU + max-pool blocks (e.g. VGG-style) that progressively halves the spatial resolution and grows the channel depth. This extracts hierarchical features.
  • Deconvolution / decoder path: a mirror stack of up-sampling (or transposed-conv) blocks that recovers the original H×W. Skip-connections from matching encoder levels (U-Net style) preserve fine boundaries.
Output: a 2-channel (or K-channel) map H×W with per-pixel softmax. The "road" channel after argmax gives the road mask.

B — SOT vs MOT compared

AspectSOT (Single-Object Tracking)MOT (Multi-Object Tracking)
Short / long-term Usually short-term — track a target manually picked in frame 1 until it leaves. Typically long-term — handle entries, exits and re-identification across the whole clip.
Detection-free / detection-based Detection-free: initialised once from a manual bounding box; tracker updates from there. Detection-based ("tracking-by-detection"): a detector runs every frame, then data-association links detections into tracks.
Type of tracking errors Drift (template appearance changes), occlusion failure, scale drift. Identity switches (ID swaps between similar objects), missed detections, fragmented tracks.
Evaluation metrics Centre-pixel error, success plot, IoU vs frame, average overlap. MOTA, MOTP, IDF1, ID switches (IDS), mostly-tracked / mostly-lost ratios.

Credit · Question 4 — Face localisation in a clip + autoencoder denoising

A — Face localisation by tracking, then re-localising every $N$ frames

"Track until the face is lost / drifts, then re-localise" = classic detection-assisted tracker. Two algorithms cooperate: a localiser (fires every $N$ frames) and a tracker (fires every frame).

A.i — The Face-localisation algorithm

  1. Proposed training dataset. Frames of the reporter labelled with the face bounding box (positives), plus other faces and background patches (negatives). Augment with rotations, scales, lighting jitters, partial occlusions.
  2. Pre-processing. Resize each frame to a fixed input size; convert to skin-colour / grayscale if the model uses it; normalise to zero-mean.
  3. Training. Train a face detector (e.g. MTCNN / Faster R-CNN / a small YOLO) end-to-end with classification + bbox-regression losses.
  4. Testing. Run the detector on every $N$-th frame; if it returns a high-confidence face, snap the box; if no detection, hand control back to the tracker.

A.ii — The Tracking algorithm (per video clip)

  1. Initialise the tracker from the most recent detector output (face bounding box).
  2. For each subsequent frame, run a single-object tracker (e.g. KCF / MOSSE / Siamese-FC) to update the box from the previous frame.
  3. Maintain a small appearance model that is refreshed only on high-confidence updates (to fight drift).
  4. Every $N$ frames, run the face-localisation algorithm; if it confidently disagrees with the tracker, re-initialise the tracker from the new box.
  5. Output trajectory = the sequence of $(x, y)$ centres (and box sizes) per frame.

B — Autoencoder for image denoising

Denoising autoencoder = "feed noisy in, ask the network to output clean." It learns to keep the structural content and throw away the noise because reconstructing the clean target is impossible if the bottleneck stores noise.
Architecture (encoder → bottleneck → decoder):
  • Encoder. Stack of conv + ReLU + pooling (or strided conv) layers that compress the noisy input H×W×3 down to a small latent code $z$.
  • Code (bottleneck). A low-dimensional representation — forces the network to keep only the structural, denoised information.
  • Decoder. Mirror stack of transposed-conv / upsampling layers that maps $z$ back up to a clean H×W×3 image.
Training. Inputs = clean images plus added noise; targets = original clean images. Loss = MSE (or perceptual) between decoder output and clean target. After training, feeding any noisy image returns its denoised version.
