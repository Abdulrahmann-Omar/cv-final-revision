SC · Question 1 — True / False (20 statements, 10 marks)

Official key, plus a one-line "why" per item

For a Gaussian, σ (the shape) matters more than the kernel size; the size is just chosen big enough to cover the bell. So statement 1 is False — and that's the only easy trap in this block.
1.FalseIt's the filter shape (σ) that defines the smoothing scale, not the kernel size. Two Gaussians with the same σ but different window sizes smooth almost identically.
2.TrueAveraging assumes independent zero-mean noise across pixels — that's why averaging N samples reduces noise by √N.
3.FalseK-means needs you to pick K up front. If you don't know K, Mean-shift (or hierarchical clustering) is the safer choice.
4.TrueNaïve 2D convolution costs O(H·W·k²): both the image size and the filter size enter.
5.TrueSobel approximates the first derivative → it highlights edges, it doesn't sharpen.
6.TrueCanny convolves with the gradient of a Gaussian — i.e. 1st-derivative of a Gaussian.
7.TrueThe given Laplacian kernel is symmetric (rotating 180° leaves it unchanged), so convolution = correlation for it.
8.TrueThe picture shows salt-and-pepper-style noise; the median filter is exactly the tool for it — it kills isolated spikes without blurring edges.
9.TrueGood descriptors (SIFT, SURF, …) are built to survive both lighting changes (photometric) and rotation/scale/affine (geometric).
10.TrueThe Laplacian is a 2nd-derivative — it amplifies noise. Smoothing with a Gaussian first → Laplacian-of-Gaussian, which is what actually works.
11.FalseThat kernel detects vertical edges (it's Sobel-X). For horizontal edges you transpose it — put the ones on top and bottom, zeros in the middle row.
12.FalseTrick wording from the key. RANSAC fits a model robustly by repeatedly sampling — it isn't a generic "outlier remover" you can apply to point correspondences without a model. (Some keys mark it True; the official key here is F.)
13.TrueLowering both Canny thresholds keeps weak edges that the hysteresis would otherwise drop → more pieces of edge survive, including broken/discontinuous ones.
14.FalseMask R-CNN is built on top of Faster R-CNN (not Fast R-CNN) — Faster gave it the RPN.
15.FalseThe main parameter of Mean-shift is the window (kernel) size / bandwidth. Window position just initialises the search; what matters is how big the window is.
16.TrueIn a Bag-of-Visual-Words classifier, the codebook size (number of visual words) is the dominant hyper-parameter.
17.TrueRecognition (1-of-N) is one specific case of verification (is this the claimed identity, yes/no) — recognition can be implemented by running verification against every gallery entry.
18.FalseIf the detector outputs a wrong class, that's a false positive for the wrong class (and a false negative for the right class). The default phrasing the key marks is "False Positive".
19.TrueYOLO predicts one (or several anchored) box(es) per grid cell — so it naturally fires multiple responses, then NMS thins them.
20.TrueU-Net's decoder uses convolutions and up-convolutions (transposed convolutions) to upsample feature maps back to full resolution.

Now think about h = g − (f ⊛ average). g = 2f. So h = 2f − blur(f) = f + (f − blur(f)). The bracket "f minus its own blur" is exactly the unsharp mask. Adding it back on top of f is what sharpening means.
h(x,y) represents a sharpening of f(x,y) — original plus its high-frequency content.

Part B — the operator is Laplacian-of-Gaussian

∇² is the Laplacian (sum of 2nd derivatives). Apply it to a Gaussian hσ and you get the Laplacian-of-Gaussian (LoG), a smoothed edge detector — same idea as a "Mexican hat" filter.

i. Name & purpose. ∇²hσ is the Laplacian-of-Gaussian (LoG). Convolving an image with it is the standard edge detector for blurry/noisy images — the Gaussian smooths first, the Laplacian fires on the 2nd-derivative.

ii. Where g(x,y) = 0? At the edges of I — specifically the zero-crossings of the second derivative. (The Laplacian peaks on either side of an edge and crosses zero at the edge.)

iii. Role of σ. σ controls the scale / width of the Gaussian. Increasing σ → more smoothing → only the largest, slowest edges survive → fewer zero-crossings. Decreasing σ → fine detail survives → many zero-crossings.

Option (a) applies the derivative after the (expensive) image convolution. Option (b) takes the derivative of the kernel first (cheap — it's just one small filter), then convolves with the image. Convolution is associative, so both give the same answer mathematically; (b) is faster.

iv. Which is preferable, (a) or (b)? (b). Because ∇² and convolution commute, you can pre-compute ∇²hσ once as a small LoG kernel and convolve the image with it only once — much cheaper than convolving with hσ first and then taking the Laplacian of the whole image.

Part D — keypoint localisation across the 4 DoG matrices

For SIFT keypoint localisation you compare each pixel against its 26 neighbours: 8 in its own scale + 9 in the scale above + 9 in the scale below. A pixel is a keypoint only if it's a strict max or min of all 26.
Only the middle two scales (scale 2 and scale 3) have neighbours both above and below, so the keypoint must live there.

SC · Question 3 — Pipelines & architectures (20 marks)

Part A — Bag-of-Visual-Words pipeline for leaf retrieval

"Handcrafted features + similarity score" is the textbook cue for Bag-of-Visual-Words (BoVW). Extract local features (SIFT / SURF), cluster them into a visual vocabulary, turn every image into a histogram, then compare histograms.
  1. Feature extraction. Run SIFT (or SURF) on every training image and the test image — you get a bag of 128-D descriptors per image.
  2. Learn the visual vocabulary. Pool descriptors from all training images and run K-means (e.g. K = 500). Each cluster centre is one "visual word".
  3. Quantise. For every descriptor in an image, assign it to its nearest visual word.
  4. Build the histogram. Count how many descriptors per word — that's a K-bin histogram = the image's BoVW representation.
  5. Normalise each histogram (L1 or L2) so image size doesn't matter.
  6. Score similarity. For each training image, compute cosine similarity (or 1 − Euclidean distance) between its histogram and the test's. The training image with the highest similarity is the match.
similarity(traini, test) = cos(hi, htest) ∈ [0, 1]. Output the training index with the maximum score.

Part B — Fast R-CNN block labels

Walk the diagram bottom-up: image → backbone → region proposals → RoI pooling → FC head → (classification, regression). That ordering is the only thing you have to remember.
#Block / arrowWhat it does
1ConvNet backboneRuns once on the whole image to produce a shared feature map (e.g. VGG/ResNet).
2Region proposals~2,000 candidate boxes generated by Selective Search (external, not learned).
3RoI PoolingCrops each proposal out of the feature map and warps it to a fixed-size grid (e.g. 7×7) so the FC layers can ingest it.

Part C — FCN architecture & the role of skip connections

FCN has the silhouette of a "backbone that downsamples → upsampling head". Conv-Conv-Pool stages shrink the resolution and grow the channels; then 1×1 conv + deconv layers blow the map back up to H × W with one channel per class.

General architecture (left → right):

  1. Input — H × W × 3 image.
  2. Encoder (down-sampling backbone) — typically VGG: repeated conv → conv → pool. Spatial size halves at each pool while feature channels grow.
  3. 1×1 conv heads at the end of each pool stage (and at the deepest feature map) convert features into a per-class score map.
  4. Decoder (up-sampling) — transposed convolutions (deconv) progressively double the resolution back to H × W.
  5. Skip connections — the per-class score maps at pool3 and pool4 are added to the upsampled deeper map (FCN-16s, FCN-8s).
  6. Output — H × W × (C + 1) pixel-wise class scores; argmax (or per-pixel softmax) gives the segmentation.

Why skip connections?

Deep pooled features know what is in the image but lose where. The shallow pool3/pool4 maps still carry sharp spatial detail. Adding them into the upsampled stream gives the FCN both semantic accuracy and crisp boundaries — without skips, segmentation masks look blocky.

Credit · Question 1 — Faster R-CNN diagram & SIFT vs HoG (25 marks)

Part A — sketch Faster R-CNN, explain "two-stage"

Faster R-CNN is the one where "the CNN does its own proposals". The whole pipeline runs in two stages: stage 1 outputs candidate boxes, stage 2 classifies + refines each candidate.

Sketch (top-to-bottom):

  1. Image → backbone CNN → shared feature map.
  2. Stage 1 — Region Proposal Network (RPN): a small conv head slides over the feature map, predicts objectness + box offsets at every anchor → ~300 region proposals.
  3. Stage 2 — Per-region head: RoI Pool/Align crops each proposal from the feature map → two FC layers → two heads (Softmax classification + bbox regression).
It's called two-stage because object detection happens in two passes: first propose objects (RPN), then classify & refine them (per-region head). Single-stage detectors (YOLO, SSD, RetinaNet) skip the proposal step and predict everything at once.

Part B — SIFT vs HoG for whole-image classification

Two simple rules of thumb:
  • SIFT = describes keypoints → great for matching the same object across two images.
  • HoG = describes the whole image as one fixed-length vector → great for classification.
HoG is the better choice. The task is image classification (label each whole image, one object per image) — HoG produces a single global descriptor per image that you can feed straight to an SVM/MLP. SIFT would give you a variable-length bag of keypoint descriptors, which then needs an extra BoVW step before classification.

Credit · Question 2 — RANSAC iterations & Harris response R (25 marks)

Part A — RANSAC pipeline + iteration count

RANSAC is "guess-and-score". Pick the minimum number of points to define the model, fit it, count how many other points agree → if better than the current best, keep it. Repeat.

Algorithm (line case):

  1. Sample 2 random points (a line needs 2).
  2. Solve for the line through those 2 points.
  3. Score: count how many of the rest are within distance ε of that line — these are inliers.
  4. Repeat N times and keep the line with the most inliers; optionally refit using all its inliers at the end.

Minimum number of iterations N:

Formula: N > log(1 − p) / log(1 − (1 − e)s), where e = outlier ratio, p = desired success probability, s = sample size (2 for a line).
Plug in: e = 1 − 0.65 = 0.35, p = 0.9, s = 2.
Numerator: log(1 − 0.9) = log(0.1) ≈ −2.303
Denominator: log(1 − (0.65)2) = log(1 − 0.4225) = log(0.5775) ≈ −0.549
N > −2.303 / −0.549 ≈ 4.19.
Round up: N = 5 iterations.

Part B — second-moment matrix and Harris response R

The Harris matrix M = [[A, B],[B, C]] = Σ over a window of [[Ix2, IxIy],[IxIy, Iy2]]. Its eigenvalues encode "two strong gradient directions → corner", "one direction → edge", "neither → flat". The Harris response shortcut avoids computing the eigenvalues directly: $$R = \det(M) - \alpha\,\text{trace}(M)^2.$$

i. What is the second-moment matrix? It's the 2×2 matrix of gradient products summed over a small window around a pixel. Diagonal entries measure squared x- and y- gradient energy; off-diagonal measures correlation. Good corners are spots where both Ix2 and Iy2 are large simultaneously — equivalently, both eigenvalues of M are large.

ii. Given M = [[403, 385],[385, 381]], α = 0.04:

det(M) = 403 × 381 − 385 × 385 = 153,543 − 148,225 = 5,318.
trace(M) = 403 + 381 = 784. trace² = 614,656.
R = 5,318 − 0.04 × 614,656 = 5,318 − 24,586.24 ≈ −19,268.24.
Interpret the sign of R: R > 0 → corner; R ≈ 0 → flat; R < 0 → edge. Our R is very negative ⇒ edge.
R ≈ −19,268.24 → the matrix corresponds to an edge, not a corner.

Credit · Question 4-B — Siamese network & triplet loss (13 marks)

Solution — Siamese architecture & triplet loss

A Siamese network has two (or three) identical CNN branches with shared weights. Each branch turns an image into an embedding f(x) ∈ ℝd. Two images of the same identity should have small ‖f(x₁) − f(x₂)‖²; different identities should have large.

Sketch (two-branch verification):

x⁽¹⁾ → CNN → f(x⁽¹⁾) ∈ ℝd
x⁽²⁾ → CNN (shared weights) → f(x⁽²⁾) ∈ ℝd
d(x⁽¹⁾, x⁽²⁾) = ‖f(x⁽¹⁾) − f(x⁽²⁾)‖²

Triplet loss — feed three images at a time

A = anchor, P = positive (same identity as A), N = negative (different identity).
L(A, P, N) = max( ‖f(A) − f(P)‖² − ‖f(A) − f(N)‖² + α, 0 )

The loss does two things at once: pull A and P together in the embedding space, and push A and N apart, with a safety margin α between them. Negative pairs that are already further than (positive distance + α) contribute 0 to the loss — only the "hard" or "semi-hard" triplets train the network.

Used for object / face verification: train the embedding once, then at test time compare any two images by Euclidean distance — same identity if d < threshold.
