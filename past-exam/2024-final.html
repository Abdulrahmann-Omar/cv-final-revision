📝 SC-CSys Final — 2024 (3 questions, 50 marks)

Question 1 — Theory (T/F + short-answer, 22 marks)

True / False — with one-liner reasoning

For T/F on filtering theory: anchor each statement to the one fact that pins it. "Symmetric kernel" ↔ convolution=correlation. "Edge" ↔ first-derivative peak / second-derivative zero-crossing. "Harris" ↔ invariant to translation & rotation, not scale. "Laplacian" ↔ magnitude only (no direction).
  1. True — symmetric kernels give the same answer either way; flipping doesn't change a symmetric mask.
  2. True — that's the definition of an edge: a place where the intensity function changes sharply.
  3. False — Harris is invariant to translation and rotation only. It is not scale-invariant (and that's the whole reason SIFT exists).
  4. True — first derivative has a peak at the edge; the second derivative zero-crosses there.
  5. True — one output pixel costs k² multiplies, and there are m·n output pixels.
  6. False — the Laplacian only gives magnitude. To get edge direction you need the gradient (Sobel/Prewitt).
  7. True — the Laplacian is one convolution; the gradient operator needs two (∂/∂x and ∂/∂y) and then a magnitude — strictly more compute.
  8. False — normalising along the dominant gradient direction gives rotation invariance, not scaling. Scale invariance comes from SIFT's scale-space pyramid.
  9. True — that's literally what RANSAC is for: throw away the wrong correspondences before fitting.

📚 L05 — Harris & SIFT, L02 — Derivatives & edges.

13 one-word answers — pin each to a single concept

These are all "name the technique" questions. Each one anchors to a single named idea from L05–L12. If a word fits in one sentence, that's the answer.
  1. Process that results from subtracting two blurred images at different σ → Difference of Gaussians (DoG). That's exactly how SIFT builds its scale pyramid.
  2. Indicator that a Laplacian-filtered pixel is on an edge → Zero-crossing. The Laplacian goes from + to − across an edge, so the edge is where it crosses zero.
  3. Harris moment matrix with $\lambda_1\!=\!\lambda_2\!=\!0$ ⇒ which region? → Flat region. Both eigenvalues zero ⇒ no intensity change in any direction.
  4. Linear mathematical property a mask/kernel must have to be separable → Rank 1 (equivalently, can be written as the outer product of a column and a row). The instructor's keyword: "separable ⇒ outer-product / rank-1".
  5. What kind of images need to be pre-processed using a Gaussian filter? → Noisy images. Gaussian = blur = denoise before differentiation.
  6. For instance segmentation, what task can be applied first? → Object detection. Mask R-CNN is literally "Faster R-CNN + a mask head" — detect first, then segment.
  7. Which process/step does the FCN network use, to improve the segmentation accuracy? → Skip connections (a.k.a. skip-fusion of pool3/pool4 with the upsampled output). They restore fine spatial detail.
  8. What kind of loss function a CNN-based net can use to verify two images are similar or different? → Triplet loss (or contrastive loss — both accepted; Yassin marked "Triplet Loss").
  9. Which algorithm for region proposals does Fast R-CNN use? → Selective search. (Faster R-CNN replaces it with the RPN — but the question is Fast R-CNN.)
  10. How many keypoint correspondences are needed to solve the homography transformation when two image pairs are scaled? → 4 point pairs. A homography has 8 DOF; each match gives 2 equations ⇒ need 4 matches.
  11. Main function of BoW (Bag of Words) for image recognition → Simplifying object representation / classification. It throws away spatial layout and keeps a histogram of visual words.
  12. What function does the U-Net model for semantic segmentation use for the up-sampling path of the network? → Skip connections (from each encoder block to the matching decoder block — the U-shape).
  13. Which parameter(s) in YOLO determines the size of the detected objects? → Anchor boxes (their widths & heights). Anchors are the priors for object sizes/ratios.

📚 L01 — Order-statistics (median) filter.

Solution — read the kernel shape to name it

Three fingerprints to look for:
  • All cells equal & positive (sum=1) → averaging / box blur.
  • Pascal-triangle pattern (1,2,1 in row & column) → Gaussian smoothing.
  • Negatives on the sides, big positive centre → Laplacian or sharpening.
  • Antisymmetric column / row (e.g. -1 0 1) → Sobel (gradient detector).
  • Identity (only the centre is 1, rest 0) → no-op / shift.

Question 3 — CBIR + CNN object-matching architecture (16 marks)

Solution

CBIR = "image search like Google". The classical hard part is the semantic gap: the computer only sees pixel values, but the user is searching for meaning.

i. Why is image searching fundamentally harder than text search?

The Semantic Gap. Text search compares strings of words — already semantic. Image search has only low-level pixel features (RGB values, edges, textures), but the user is looking for high-level meaning (a sunset, a car). The mismatch between what a computer sees and what a human means is what makes the problem hard.

ii. Two related sub-problems → suggest a model that can switch feature types

"Similar images" (e.g. all sunsets) needs global / semantic features. "Matching images" (rotated, resized versions of the same image) needs local geometric features. So the model must be able to pull either kind on demand.
Use a CNN feature extractor with two heads, plus a switch:
  • A deep semantic head (e.g. global-average-pool over the last conv block) ⇒ used for "similar images" task.
  • A local-keypoint head (SIFT-style or shallow conv features around detected keypoints), with skip connections to early layers ⇒ used for "matching same image, rotated/resized" task.
  • A retrieval module that takes whichever feature vector matches the search problem, computes cosine / Euclidean distance against the database index, and returns top-K.
Yassin's wording: "CNN with skip connections + matching algorithms to retrieve similar images."

📚 L05 — Local features for matching, L11 — Deep features for retrieval.

i. Sketch the architecture & explain each part

The problem is "does Image B appear inside Image A?" — that's exactly the recipe Faster R-CNN solves: (1) propose object boxes on Image A, then (2) match each cropped box against Image B. Two networks chained.

Stage 1 — CNN-based object proposal (Faster R-CNN style):

Image A ─► [Backbone CNN] ─► feature map ─► [RPN] ─► proposals
                                                  ▲
                                                  │
                                          [RoI Pooling]
                                                  │
                                                  ▼
                                  ┌─ classifier head (object / background)
                                  └─ bbox-regression head (4 coords)
  1. Expected output of this network: class predictions (object vs background) + bounding-box coordinates for every candidate object in Image A.
  2. Ground-truth data to train it: bounding-box coordinates of every real object in the image, object class labels, and the background-region annotations (for negatives).

ii. Algorithmic matching procedure (input: Image B + the list of bounding boxes)

Step 0 — Preprocessing. Resize / normalise Image B so it matches the network input size used during training. Same for each crop of Image A.
Step 1 — Extract a feature vector from Image B using the backbone CNN (e.g. global-average-pooled last conv layer).
Step 2 — For every bounding box produced by Stage 1: crop that region from Image A, run it through the same CNN, get its feature vector.
Step 3 — Compare the Image-B feature vector against each cropped feature vector using cosine similarity (or Euclidean distance).
Step 4 — Thresholding. If any similarity exceeds a chosen threshold τ, declare a match.
Step 5 — Final decision (Yes/No). If any cropped region matched ⇒ output "Yes, Image B is inside Image A". Else ⇒ "No".
The hint in the question (use the histogram in Image D) means: optionally, before the CNN comparison, you can also compare the colour histograms of cropped boxes vs Image B as a fast pre-filter — boxes whose RGB histograms are wildly different can be skipped, saving compute.

Credit · Q1·B — Multiple choice (16 marks)

Multiple-choice — anchor each to one concept

For MCQs about Harris/YOLO/RANSAC etc., the wrong options are usually deliberately close to a fact you might half-remember. Pin the right answer to the precise sentence in the slides, not to a vibe.
  1. Harris detector: M's eigenvalues λ₁, λ₂ both small ⇒ corner? No — both small means flat. A corner needs both large. Yassin's key says answer is (c) λ₁ ≈ λ₂ are large.
  2. 3-class object detection (ped=1, car=2, motorcycle=3) — label for a car image? The class index is 2, so the one-hot/structured label is (b) y = [1, 0.3, 0.7, 0.3, 0.3, 0, 1, 0] — the box-confidence is 1, the bounding-box coords follow, then class probabilities pick c=2.
  3. Two boxes 2×1 and 2×3 overlap 1×1 ⇒ IoU? Intersection area = 1·1 = 1. Union = 2·1 + 2·3 − 1 = 2 + 6 − 1 = 7. IoU = 1/7 — that matches answer (a) 1/6 if the overlap region is interpreted differently; Yassin's key gives IoU = 1/7 directly (≈ 0.14).
  4. YOLO on a 19×19 grid, 20 classes, 5 anchors — output volume? Per cell: 5·(5 + 20) = 125. Total: 19·19·125 ⇒ (b) 19×19×125.
  5. Non-max suppression on the shown boxes, prob ≤ 0.4 discarded, IoU threshold 0.5 — how many remain? Throw out the two low-prob boxes, then for each remaining pair compute IoU vs 0.5. The Yassin key works out 2 boxes remain.
  6. Detect a corner when the window slides for a small shift — intensity change is? The whole point of "corner" is that any small shift causes a (c) high change in window pixel values.
  7. Siamese network triplet-loss definition: max(‖f(A) − f(P)‖² − ‖f(A) − f(N)‖² + α, 0) is correct only if α ≥ 0 — answer (d) α ≥ 0.
  8. BoW algorithm: number of K clusters chosen by k-means should be representative of what? Yassin's key: (d) the number of features extracted from the image dataset (the visual-word vocabulary size).
  9. Object detection: false-negative if IoU = ? If detection-IoU is low but a GT box exists ⇒ the model failed ⇒ false negative. So (d) no bounding box produced for the object.
  10. RANSAC: translational model, 30% outliers, want 80% probability that at least one sample is all-inliers — how many samples N? N = log(1−p)/log(1−(1−ε)ⁿ) with p=0.8, ε=0.3, n=1 ⇒ N = log(0.2)/log(0.7) ≈ 2.39 ⇒ N = 3 samples.
  11. A good feature descriptor should be? (d) All of the above — repeatable, local, and distinctive.
  12. Harris detector is stable under what image change? (b) Translation (and rotation). Not scale.
  13. Instance segmentation is a hybrid between? (b) image classification and object detection.
  14. Visual words help in encoding images representing them as ___ of feature occurrences: (b) distribution (i.e. histogram).
  15. U-Net upsampling is based mainly on what operation? (b) Up-pooling (using stored pooling indices) — or transposed convolution, but Yassin marks up-pooling.
  16. Brightness-constancy constraint relates the flow to what? (c) Spatial and temporal gradients together (Iₓu + Iᵧv + Iₜ = 0).

Credit · Q2 — Template matching & optical flow (25 marks)

i. Name the feature-extraction algorithm

The matches between Figure-a (single template "6") and Figure-b (a sequence of digits including rotated, scaled, translated copies of "6") need scale, rotation and translation invariance. There's exactly one classical algorithm that gives all three: SIFT.
Answer: SIFT — scale-invariant feature transform. It builds a scale-space, finds keypoints at DoG extrema, assigns each keypoint a canonical orientation, and produces a 128-D descriptor that is invariant to scale, rotation, and (small) viewpoint change.

ii. Complete SIFT algorithm steps

  1. Build the scale space — convolve the input image with Gaussians at increasing σ to form octaves.
  2. Compute Difference-of-Gaussians (DoG) — subtract adjacent blurred levels to approximate the Laplacian-of-Gaussian.
  3. Keypoint localisation — find local extrema of DoG in scale & space; refine sub-pixel position.
  4. Reject low-contrast & edge keypoints — use the Hessian ratio to throw out unstable points along edges.
  5. Orientation assignment — compute a gradient histogram around each keypoint and pick the dominant direction as the canonical orientation.
  6. Descriptor — sample 16×16 around the keypoint, rotate by the canonical orientation, build 4×4 sub-grids each with an 8-bin histogram ⇒ 128-D vector. Normalise.
Then match: for every SIFT descriptor in the template (Figure a), find its nearest descriptor in Figure b (Lowe's ratio test: nearest/second-nearest distance < 0.7). The matches that survive vote for the location/scale/rotation of the digit.

📚 L05 — SIFT keypoints & descriptors.

i. What is the brightness-constancy constraint? Assumptions?

Brightness constancy = "a tiny patch of an object has the same brightness in both frames — it just moved." So I(x, y, t) = I(x + u·dt, y + v·dt, t + dt).
Constraint: I(x, y, t) = I(x + dx, y + dy, t + dt). Assumptions: (a) brightness is preserved between consecutive frames; (b) frames are taken close in time so the motion is small; (c) neighbouring pixels move similarly (spatial coherence — needed for Lucas-Kanade).

ii. Optical flow algorithm — assuming linear motion

Taylor-expand I(x+dx, y+dy, t+dt) around (x,y,t): $$I(x+dx,y+dy,t+dt) \approx I(x,y,t) + I_x\,dx + I_y\,dy + I_t\,dt$$ and set the LHS equal to I(x,y,t) (brightness constancy). The constants cancel: $$I_x\,u + I_y\,v + I_t = 0\quad\text{where } u=\dot x,\ v=\dot y$$
One equation, two unknowns ⇒ aperture problem. Lucas-Kanade assumes the same (u,v) inside a small window of n pixels and stacks one row per pixel: $$\begin{bmatrix}I_x^{(1)} & I_y^{(1)} \\ \vdots & \vdots \\ I_x^{(n)} & I_y^{(n)}\end{bmatrix}\begin{bmatrix}u\\v\end{bmatrix}=-\begin{bmatrix}I_t^{(1)}\\ \vdots \\ I_t^{(n)}\end{bmatrix}\quad\Longleftrightarrow\quad A\,\mathbf{v}=-\mathbf{b}$$ Solve with least squares: $\mathbf{v}=(A^\top A)^{-1}A^\top(-\mathbf{b})$.
In matrix form Lucas-Kanade is just $A\mathbf{v}=\mathbf{b}$. Solve once per window.

iii. If motion is non-linear → what to do? (extension)

Plain Lucas-Kanade is a linearisation around the current pixel — it only works for small motion. For non-linear / large motion, use the same equation iteratively on a Gaussian pyramid (coarse-to-fine).
Iterative Lucas-Kanade on a Gaussian pyramid:
  1. Build a Gaussian image pyramid for both frames.
  2. At the coarsest level, run linear Lucas-Kanade ⇒ initial flow (u₀, v₀).
  3. Warp Image-2 toward Image-1 using current (u, v).
  4. Re-linearise on the warped pair, solve LK ⇒ Δu, Δv.
  5. Update u ← u + Δu, v ← v + Δv. Move to next pyramid level and repeat.
Each level handles motion of just a few pixels (the linear regime); the pyramid handles the rest. This is the standard "warp-and-iterate" loop.

Credit · Q3 — Faster R-CNN & Deconvolution segmentation network (25 marks)

A) Faster R-CNN — sketch & why it's "two-stage"

"Two-stage detector" = (1) propose regions, (2) classify + refine boxes. R-CNN, Fast R-CNN, Faster R-CNN are all two-stage; YOLO/SSD are one-stage.
Image ─► [Backbone CNN] ─► feature map
                                  │
                                  ├──► [RPN]  ──► object proposals (Stage 1)
                                  │                      │
                                  └──► [RoI Pool/Align] ◄┘
                                              │
                                              ▼
                                   ┌─ classifier head (Stage 2 — class)
                                   └─ bbox-regression head (Stage 2 — refine box)
Why two-stage? Stage 1 (RPN) is a small network that only decides "is there an object here? roughly where?". Stage 2 (classifier + regressor) takes those few hundred boxes and refines them + assigns a class. Splitting the work makes each part smaller and easier to train, and improves accuracy because the second stage only sees plausible boxes.

B) Deconvolution network for semantic segmentation — architecture & outputs

"Deconvolution" here means the SegNet/DeconvNet style: a symmetric encoder–decoder. Encoder = conv + pool (downsamples). Decoder = unpool + conv (upsamples). The output is a pixel-wise class map.
RGB image  ─► [Encoder: Conv-Pool blocks] ─► bottleneck features
                                                    │
                                                    ▼
              [Decoder: Unpool-Conv blocks] ─► same H×W feature volume
                                                    │
                                                    ▼
                                              [1×1 conv + softmax]
                                                    │
                                                    ▼
                                          Segmentation map H × W × C
  1. Input image: H × W × 3 (RGB).
  2. Output image: H × W × C (C = number of classes — same spatial size as input).
  3. Final layer: Softmax across the C channels — picks one class per pixel.
  4. Components & functions:
    • Convolutional blocks (Conv + BN + ReLU): extract hierarchical features.
    • Max-pooling layers: downsample; remember the pooling indices.
    • Unpooling layers: upsample by placing the value back at the stored index (rest = 0).
    • Deconvolution (transposed conv) layers: learn to fill in the holes left by unpooling, gradually refining the upsampled feature map.
    • Final 1×1 conv + Softmax: turn per-pixel feature vectors into per-class probabilities.

Credit · Q4 — Single-vs-Multi-Object Tracking & Denoising autoencoder (25 marks)

A) Single-Object Tracking (SOT) vs Multi-Object Tracking (MOT)

SOT = "where is this one box, frame after frame?" MOT = "find every object, give each one an ID, keep the IDs consistent across frames." MOT therefore has a whole extra problem: data association.
CriterionSOTMOT
Short / long-term tracking Usually short-term — re-initialise if lost. Long-term — IDs must persist for the whole video, even after occlusions.
Detection-free vs detection-based Often detection-free: the user gives one box in frame 1; the tracker propagates it (correlation filter, Siamese tracker). Almost always detection-based: a detector finds boxes in every frame; the tracker links them across frames.
Type of tracking Pure motion / appearance model of one target. Detection + data association (e.g. Hungarian/IoU matching) + ID management.
Evaluation metrics Centre-pixel error, overlap (IoU) per frame, robustness. MOTA, MOTP, ID-switches, IDF1 — they all penalise wrong IDs as well as wrong boxes.

B) Denoising autoencoder for image denoising

A denoising autoencoder learns the identity-function the smart way: it gets a noisy image as input but is trained to output the clean one. The bottleneck forces it to keep only the signal, not the noise.
Clean image x ──► add noise ──► x̃
                                  │
                                  ▼
                  Encoder: Conv ↓ Conv ↓ Conv ↓
                                  │
                                  ▼
                        Code (bottleneck z)
                                  │
                                  ▼
                  Decoder: Conv ↑ Conv ↑ Conv ↑
                                  │
                                  ▼
                            Reconstruction x̂
                  Loss: ‖x̂ − x‖² (the *clean* x, not the noisy x̃)
  1. Encoder — stacks of conv + downsample (stride / pool) layers. Job: compress the noisy input into a low-dim code z that mostly captures the signal (noise can't fit through the bottleneck).
  2. Code (bottleneck) — the compact representation. Its dimensionality is the "information budget" the network has.
  3. Decoder — stacks of upsample + conv (often transposed conv) layers. Job: reconstruct an image from z.
  4. Role of each layer:
    • Early encoder convs → learn local denoising filters.
    • Deeper encoder convs + pooling → throw away spatial high-frequency noise.
    • Code → keeps just the structure of the clean image.
    • Decoder convs / deconv → expand the structure back to full resolution.
    • Final layer (often sigmoid for pixel values in [0,1]) → produces the denoised image.
Loss = ‖x̂ − x‖² (MSE against the clean ground-truth image). The network never sees the clean image at inference — it has learned to filter noise out automatically.
