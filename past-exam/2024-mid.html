📝 2024 Midterm

Question A — Eight True / False statements (8 marks)

Solution — verdict per statement

For Gaussian / Laplacian / Sobel / Canny statements, anchor yourself on three rules:
  • For a Gaussian, the σ (not the kernel size) is what controls the smoothing scale — the size only needs to be big enough to hold ≈ ±3σ.
  • A Sobel kernel has positive on one side, zero in the middle, negative on the other side. The zero column tells you which gradient direction is being measured: zero in the x-column → horizontal change → vertical-edge detector; zero in the y-row → vertical change → horizontal-edge detector.
  • Canny uses the 1st derivative of a Gaussian, Marr–Hildreth uses the 2nd derivative (LoG). Don't confuse them.

1Gaussian filter size is the most important parameter (it defines the smoothing scale).FalseIt's the σ that defines the smoothing scale. The size just has to be wide enough (~6σ) to hold the bell.

Pattern of answers: 1·F, 2·T, 3·T, 4·T, 5·T, 6·F, 7·F, 8·F.

Question B — 1D second-derivative convolution (3 marks)

Solution

The kernel [-1, 2, -1] is the discrete second derivative (Laplacian in 1D). It outputs zero on any flat region (the two −1's cancel the centre 2·v), a positive spike just before a step up, and a negative spike just after. That's why you only get non-zero values right at the edges of the box.

Step 1 — slide the kernel and compute each output

For convolution, you flip the kernel before sliding. [-1, 2, -1] flipped is itself (it's symmetric), so the operation is the same as correlation. At each output position i: out[i] = −input[i−1] + 2·input[i] − input[i+1].

Worked sample at i = 3: −input[2] + 2·input[3] − input[4] = −0 + 0 − 5 = −5. At i = 4: −0 + 2·5 − 5 = 5. Inside the plateau (i = 5–7) every neighbourhood is [5,5,5] → output = −5 + 10 − 5 = 0.

Output row: 0, 0, 0, −5, 5, 0, 0, 0, 5, −5, 0, 0, 0 — a positive/negative pair at each edge of the step, zeros everywhere else.

Step 2 — would correlation give a different answer?

Correlation skips the flip step. So convolution and correlation differ only when the kernel is asymmetric. [-1, 2, -1] reads the same forwards and backwards → it's symmetric → flipping changes nothing.
No — the result is identical, because the kernel [-1, 2, -1] is symmetric, so flipping it (which is the only step that distinguishes convolution from correlation) leaves it unchanged.

Question C — Harris detector: which pair, and is M a corner? (4 marks)

Part i — Which image pair is appropriate for Harris?

Harris is invariant to rotation and to (small) intensity change, but it is NOT invariant to scale. So look at what changes between the two pictures:
  • If one image is a rotated version of the other → Harris is fine.
  • If one image is a zoomed-in / zoomed-out version → Harris fails, you need SIFT (scale-invariant).
In Pair 1 the second kitten image is rotated relative to the first. In Pair 2 the horse is at a different scale (zoom).
Pair 1, because Harris is invariant to rotation but not to scale, and Pair 1 differs only by rotation. Pair 2 needs a scale-invariant detector (SIFT) instead.

Part ii — Does M = [[29, 97], [97, 353]] correspond to a corner?

Read the diagonal of the Harris moment matrix M = [[Σ Iₓ², Σ Iₓ I_y], [Σ Iₓ I_y, Σ I_y²]]. A corner needs both eigenvalues large, i.e. Σ Iₓ² and Σ I_y² should be comparable. An edge has one diagonal entry much larger than the other (gradient lives along only one direction).
Here Σ Iₓ² = 29 while Σ I_y² = 353. The y-direction gradient dominates by more than 10×, so the image patch has strong intensity change in y but almost none in x — that's the signature of a horizontal edge, not a corner.

Cross-check with eigenvalues: trace = 29 + 353 = 382, det = 29·353 − 97·97 = 10,237 − 9,409 = 828. So λ ≈ (382 ± √(382² − 4·828))/2 ≈ (382 ± √142,612)/2 ≈ (382 ± 377.6)/2 → λ₁ ≈ 379.8, λ₂ ≈ 2.2. One huge, one tiny → edge response, exactly matching the I_y ≫ Iₓ reasoning.

No, it does not correspond to a corner — because I_y ≫ Iₓ (353 vs 29). One eigenvalue is large, the other near zero, so Harris would label this patch an edge, not a corner.
