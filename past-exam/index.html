Midterm · Question 1 — Identify what each 3×3 mask does

Solution

Matrix 1 — centre $+4$, four cardinal neighbours $-1$, corners $0$.
Apply to a pixel $f(x,y)$: $$g(x,y) = 4f(x,y) - f(x{+}1,y) - f(x{-}1,y) - f(x,y{+}1) - f(x,y{-}1)$$ This is exactly $-\nabla^2 f$ (the Laplacian) — its output is large in magnitude only at intensity changes.
Matrix 1 → Laplacian edge detector. Output ≈ 0 in flat regions, large at edges.
Matrix 2 — same as Matrix 1 but the centre is $+5$ (one bigger). That is: $$\text{Matrix 2} = \underbrace{\text{Identity}}_{\text{centre}=1} + \underbrace{\text{Matrix 1}}_{\text{centre}=4}$$ Convolution is linear, so applying Matrix 2 = applying the identity (keep the original) + applying the Laplacian. $$g(x,y) = f(x,y) + (-\nabla^2 f)$$
Matrix 2 → Sharpening mask. Original image with edges emphasised — keeps the shapes and highlights edges.

Midterm · Question 3 — Negative & γ on pixels 10, 20, 100, 200

Solution

Formulas.
  • Negative: $s = 255 - r$
  • Gamma: normalise $r \to r/255 \in [0,1]$, then $s = 255 \cdot (r/255)^{\gamma}$.

$r/255 = 100/255 = 0.392$
$0.392^{0.2}$: take logs, $0.2 \ln 0.392 = 0.2\cdot(-0.936) = -0.187$, $e^{-0.187} = 0.829$
$s = 255 \cdot 0.829 = \mathbf{211.4}$
Sample working (r = 100, γ = 2):
$0.392^{2} = 0.1538$
$s = 255 \cdot 0.1538 = \mathbf{39.2}$
Effect of increasing γ. Because $r/255 \le 1$ for every pixel, $(r/255)^{\gamma}$ decreases as γ grows. So:
  • γ > 1 → image gets darker (rows above: γ=2 column).
  • γ < 1 → image gets brighter (γ=0.2 column).
Hence the answer key's note: "γ increases ⇒ brightness decreases, and vice versa."

Solution · Part 2 — How do we make the 3rd image from the first two?

Image 1 = original $A$. Image 2 = its erosion $A \ominus B$. Image 3 = outline.
Outline = original minus its erosion: $$\boxed{\;\beta(A) = A - (A \ominus B)\;}$$ i.e. the morphological boundary-extraction operator.

📚 L03 — Boundary extraction.

Solution · Part 3 — Multiplying $f(x,y)$ by $(-1)^{x+y}$

$(-1)^{x+y} = e^{j\pi(x+y)} = e^{j\pi x} e^{j\pi y}$. Multiplying a signal by $e^{j\pi x}$ in the spatial domain shifts its DFT by $M/2$ in $u$ (and the same in $v$). So: $$f(x,y)\cdot(-1)^{x+y} \;\;\longleftrightarrow\;\; F(u-M/2,\, v-N/2)$$
Result: the DC term (originally at $(0,0)$) moves to the centre of the spectrum — the classic fftshift trick used to visualise an image's Fourier transform.

Midterm · Question 5 — SIFT

1. Why do we use rotation invariance?

We want the same physical feature to produce the same descriptor no matter how the camera is rotated. Without rotation invariance you would need a separate template at every possible angle — completely impractical for matching across viewpoints.
SIFT achieves it by assigning each keypoint a canonical orientation (peak of a 36-bin gradient-orientation histogram), then computing the descriptor in coordinates rotated to that orientation.

2. Why apply smoothing before filtering (derivatives)?

Differentiation amplifies high-frequency content. Noise is high frequency. So a naive Laplacian on a noisy image responds to noise instead of real edges.
Apply a Gaussian smoothing first to attenuate noise. The combined operator is the Laplacian-of-Gaussian (LoG) — which DoG approximates cheaply.

3. The scale-space / DoG figure

Build a Gaussian pyramid: blur the image with $\sigma, s\sigma, s^2\sigma, s^3\sigma,\dots$ Adjacent differences give the DoG: $$\mathrm{DoG}(x,y,\sigma) \;\approx\; (s-1)\,\sigma^2\,\nabla^2 S(x,y,\sigma)$$ (the scale-normalised Laplacian-of-Gaussian, NLoG).
Each DoG band $N_{s^k\sigma}$ in the figure highlights blob-like structures at a specific scale. Keypoints are extrema across 26 neighbours (8 same scale + 9 above + 9 below).

Final · Question 1 — Compare FCN and Mask R-CNN

Solution

FCNMask R-CNN
Type Semantic segmentation — every pixel gets a class label, but two cars share the "car" label (they are not separated). Instance segmentation — every detected object instance gets its own binary mask. Two cars → two masks.
Output structure A single tensor of shape H×W×C (C = number of classes). One score map per class, full image resolution. Per RoI: (class label, refined bounding box, mask tensor of shape K×m×m, e.g. K=80 classes × 28×28 mask).
Output-layer function Softmax across classes, applied per pixel (classes are mutually exclusive). Sigmoid per pixel of each class channel — class & mask are decoupled. Only the GT-class channel contributes to the mask loss.

Final · Question 2 — Write the YOLO (v2+) loss function

Solution — three terms summed

LossFormula
1. Localisation (box regression) $\displaystyle L_\text{loc} = \lambda_\text{coord}\sum_{i,j}\mathbb{1}_{ij}^{obj}\Big[(t_x{-}\hat t_x)^2 {+} (t_y{-}\hat t_y)^2 {+}(t_w{-}\hat t_w)^2 {+} (t_h{-}\hat t_h)^2\Big]$ — MSE on the parameterised offsets (Smooth-L1 from v3+).
2. Objectness (confidence) $\displaystyle L_\text{obj} = -\sum_{i,j}\Big[\mathbb{1}_{ij}^{obj}\log\hat C_{ij} + \lambda_\text{noobj}\,\mathbb{1}_{ij}^{noobj}\log(1{-}\hat C_{ij})\Big]$ — binary CE; $\hat C_{ij}$ is trained to equal IoU with the GT.
3. Classification $\displaystyle L_\text{cls} = -\sum_{i,j}\mathbb{1}_{ij}^{obj}\sum_{c}\big[y_c\log\hat p_c + (1{-}y_c)\log(1{-}\hat p_c)\big]$ — independent binary CE per class in v3+ (so multi-label is possible).
$L = L_\text{loc} + L_\text{obj} + L_\text{cls}$. The indicator $\mathbb{1}_{ij}^{obj}$ is 1 only for the "responsible" (cell, anchor); $\lambda_\text{coord}\!=\!5$ and $\lambda_\text{noobj}\!=\!0.5$ are the standard weightings.

Final · Question 3 — GIoU vs IoU vs DIoU

1. Advantage of GIoU over IoU

Problem with IoU. When the predicted box and the GT do not overlap, $\mathrm{IoU} = 0$ for any non-overlapping arrangement — the loss is constant, the gradient is zero, and the optimiser doesn't know which direction to move the box.
GIoU adds an enclosing-box penalty. Let $C$ be the smallest axis-aligned box that contains both predictions: $$\mathrm{GIoU} \;=\; \mathrm{IoU} \;-\; \frac{|C \setminus (A \cup B)|}{|C|}$$ The second term is positive whenever the boxes are far apart, so the gradient is non-zero even when the boxes don't overlap.
Boxes can be pulled together from any starting configuration. $\mathrm{GIoU}\in[-1,1]$ (vs $\mathrm{IoU}\in[0,1]$).

2. Advantage of DIoU over GIoU

GIoU still converges slowly when the boxes have the same aspect ratio and just need translation — the enclosing-box term can plateau because $|C|$ stops shrinking quickly.
DIoU directly penalises the distance between box centres: $$\mathrm{DIoU} = \mathrm{IoU} - \frac{\rho^2(b,b^{gt})}{c^2}$$ $\rho$ = Euclidean distance between centres; $c$ = diagonal of the smallest enclosing box. The penalty falls off only when centres line up.
DIoU gives a strong, well-conditioned gradient → faster convergence and better localisation. (CIoU adds an aspect-ratio term on top.)

Final · Question 4 — R-CNN family & IoU loss

1. Two reasons Fast R-CNN is faster than R-CNN

(a) Single CNN pass over the image. R-CNN warps each of ~2,000 proposals to a fixed size and runs the full CNN backbone on every one — heavily redundant for overlapping patches. Fast R-CNN runs the backbone once on the whole image and reuses the feature map; per-proposal cost shrinks to RoI Pool + a small FC head.
(b) Single end-to-end multi-task training. R-CNN has three stages (CNN finetune → per-class SVMs → bbox regressors) with intermediate features cached on disk. Fast R-CNN replaces all of that with one network trained jointly under $L = L_\text{cls} + \lambda L_\text{box}$.
Combined: ≈ 9× faster training and ≈ 100× faster inference vs R-CNN (paper's numbers).

2. Why use $1 - \mathrm{IoU}$ instead of $\mathrm{IoU}$?

Training procedures minimise a loss. $\mathrm{IoU}=1$ is the ideal (boxes match perfectly) and $\mathrm{IoU}=0$ is the worst — that's the wrong direction for a loss.
Flipping with $1-\mathrm{IoU}$ makes the perfect overlap correspond to a loss of $\boxed{0}$, which is the convention every optimiser expects.

Final · Question 6 — Output vector size on a 15×15 grid (80 classes, 10 anchors)

Solution

General formula. Each anchor predicts $(p_c, b_x, b_y, b_w, b_h) = 5$ box values plus class scores. With $M$ active anchors and $N$ classes:
  • Per-anchor classes (YOLO v2+): per-cell length = $M\cdot(5+N)$
  • Shared classes (YOLO v1): per-cell length = $M\cdot 5 + N$

1. Only one object per cell ⇒ M = 1

Per-cell length = $1\cdot(5+80) = \boxed{85}$. Output tensor $15\times 15\times 85 = 19{,}125$ values.

2. Up to 10 objects per cell ⇒ M = 10

Per-anchor classes: $10\cdot(5+80) = \boxed{850}$ per cell, tensor $15\times 15\times 850 = 191{,}250$ values.
Shared classes: $10\cdot 5 + 80 = \boxed{130}$ per cell, tensor $15\times 15\times 130 = 29{,}250$ values.

Comment

Per-anchor classes use ≈ 10× more parameters but allow a single cell to predict objects of different classes. Shared classes are cheaper but force every anchor of a cell to commit to the same class — that's why YOLO v2 switched to per-anchor.

Final · Question 7 — Choosing the correct number of anchor boxes for a YOLO dataset

Solution — K-means on box dimensions

  1. Collect all GT bounding-box dimensions $(w,h)$ from the training set.
  2. Run K-means clustering for $K = 3,4,5,6,7,8,9,\dots$ on those $(w,h)$ pairs.
    Important: use $1 - \mathrm{IoU}$ as the distance metric (not Euclidean) so a 10×100 box doesn't get clustered with a 100×10 box just because their hypotenuse matches.
  3. For each $K$: assign every GT box to its nearest cluster centroid (the cluster centre is an anchor). Compute the mean IoU between every GT and its assigned anchor.
  4. Plot Mean IoU vs K — it rises monotonically and saturates.
  5. Pick the smallest K whose mean IoU exceeds an acceptable threshold (typical: ≥ 0.6). Classic elbow trade-off: more anchors → more parameters, slower inference; fewer anchors → worse fits.
This is exactly the "Dimension Clusters" procedure introduced in YOLO v2 — it justifies why the default is $k = 5$ (mean IoU ≈ 0.61 on COCO) rather than picking the 9 anchors of Faster R-CNN.

Final · Question 8 — Bilinear interpolation at $(x,y) = (4, 7)$

Solution

Known corners. $f(3,5) = 15$, $f(9,5) = 35$, $f(3,8) = 40$, $f(9,8) = 20$.
Target. $f(4, 7) = ?$
Step 1 — compute the normalised offsets a, b. $$a = \frac{x - x_1}{x_2 - x_1} = \frac{4 - 3}{9 - 3} = \frac{1}{6} \approx 0.167$$ $$b = \frac{y - y_1}{y_2 - y_1} = \frac{7 - 5}{8 - 5} = \frac{2}{3} \approx 0.667$$
Step 2 — apply the bilinear formula. $$f(x,y) = (1-a)(1-b)\,f(3,5) + a(1-b)\,f(9,5) + (1-a)b\,f(3,8) + ab\,f(9,8)$$

Weights:

CornerWeightValueContribution
(3, 5) = 15$\tfrac{5}{6}\cdot\tfrac{1}{3} = \tfrac{5}{18}$15$\tfrac{75}{18}$
(9, 5) = 35$\tfrac{1}{6}\cdot\tfrac{1}{3} = \tfrac{1}{18}$35$\tfrac{35}{18}$
(3, 8) = 40$\tfrac{5}{6}\cdot\tfrac{2}{3} = \tfrac{10}{18}$40$\tfrac{400}{18}$
(9, 8) = 20$\tfrac{1}{6}\cdot\tfrac{2}{3} = \tfrac{2}{18}$20$\tfrac{40}{18}$
Sum$\tfrac{550}{18}$
Step 3 — final value. $$f(4, 7) = \frac{550}{18} = 30.\overline{5} \;\approx\; \boxed{30.56}$$
Sanity check: $a$ is small (the target is near the left edge), so the left-column corners (15 and 40) should dominate — and indeed weight 5/18 + 10/18 = 15/18 ≈ 83 % of the total. $b$ is large (target near the bottom), so the bottom corners (40 and 20) get more weight than the top — bottom weight = 10/18 + 2/18 = 12/18 ≈ 67 %. Both match the figure.
